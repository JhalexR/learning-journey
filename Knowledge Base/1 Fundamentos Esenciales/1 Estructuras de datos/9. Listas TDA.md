# ¿Qué es una lista?

Una lista es un Tipo de Dato Abstracto (TDA) que representa una colección ordenada de elementos. 

Cada elemento ocupa una posición dentro de la colección y puede accederse mediante su índice o recorriendo la lista.

**Ejemplo conceptual**

Imagina una lista de tareas:

```
[Estudiar, Hacer ejercicio, Leer, Dormir]
```

Cada elemento tiene una posición:

| Índice | Elemento        |
| -----: | --------------- |
|      0 | Estudiar        |
|      1 | Hacer ejercicio |
|      2 | Leer            |
|      3 | Dormir          |

> **_El orden es importante. Si cambias el orden -> ya no es la misma lista._**

### Características principales

+ **Ordenada:** los elementos tienen una secuencia definida.
+ **Dinámica:** normalmente puede crecer o disminuir.
+ _Permite elementos repetidos._ => [5, 8, 5, 2, 8]

### Operaciones básicas

| Operación | Descripción                               |
| --------- | ----------------------------------------- |
| Crear     | Construir una lista vacía.                |
| Insertar  | Agregar un elemento.                      |
| Eliminar  | Quitar un elemento.                       |
| Buscar    | Encontrar un elemento.                    |
| Obtener   | Acceder al elemento en una posición.      |
| Modificar | Cambiar el valor de un elemento.          |
| Recorrer  | Visitar todos los elementos.              |
| Tamaño    | Saber cuántos elementos hay.              |
| Vacía     | Verificar si la lista no tiene elementos. |

### Idea clave

Puedes pensar en una lista como una colección ordenada que responde a estas preguntas:

+ ¿Qué elementos contiene?
+ ¿En qué orden están?
+ ¿Cómo puedo agregar, eliminar, buscar o recorrer elementos?

**_Sin preocuparte todavía por cómo está construida internamente._**

## Implementaciones

Existen dos implementaciones principales:

1. Lista secuencial (basada en arreglos).
2. Lista enlazada (basada en nodos y referencias).

```
          TDA Lista
              │
     ┌────────┴────────┐
     │                 │
Lista secuencial   Lista enlazada
 (arreglo)              │
               ┌────────┴─────────┐
               │                  │
        Simplemente          Doblemente
         enlazada             enlazada
```

### 1. Implementación mediante arreglos (Lista secuencial)

En esta implementación, los elementos se almacenan en posiciones contiguas de memoria.

```
Índice:   0    1    2    3
         ┌───┬───┬───┬───┐
Arreglo: │10 │20 │30 │40 │
         └───┴───┴───┴───┘
```

#### ¿Cómo se accede?

Como conocemos la posición de cada elemento, acceder es muy rápido.
El computador calcula directamente la dirección de memoria.

```
lista[2] = 30
```
#### Inserción

Supongamos que queremos insertar 25 entre 20 y 30.
Se deben mover todos los elementos hacia la derecha.

```
            ------>
┌───┬───┬───┬───┬───┐
│10 │20 │25 │30 │40 │
└───┴───┴───┴───┴───┘
```

#### Eliminación

Eliminar 20 implica mover todos los elementos posteriores.

```
    <------
┌───┬───┬───┬───┐
│10 │25 │30 │40 │
└───┴───┴───┴───┴
```

#### Ventajas
+ Acceso muy rápido por índice.
+ Poco consumo de memoria adicional.
+ Excelente aprovechamiento de la memoria caché del procesador.

#### Desventajas
+ Insertar puede ser costoso.
+ Eliminar puede ser costoso.
+ Si el arreglo tiene tamaño fijo, puede quedarse sin espacio.

### 2. Implementación mediante listas enlazadas

Aquí los elementos no están uno junto al otro en memoria. Cada elemento se almacena en un nodo.

Cada nodo contiene:

+ el dato
+ una referencia al siguiente nodo

La lista anterior que representabamos así con arreglos:

```
Índice:   0    1    2    3
         ┌───┬───┬───┬───┐
Arreglo: │10 │20 │30 │40 │
         └───┴───┴───┴───┘
```
Ahora se representaria así con listas enlazadas:

```
┌───────┐
│Cabeza │
└───┬───┘
    │
    ▼
┌──────────┐
│10 |  ●───┼────►
└──────────┘
              ▼
        ┌──────────┐
        │20 | ●────┼────►
        └──────────┘
                    ▼
              ┌──────────┐
              │30 | ●────┼────►
              └──────────┘
                          ▼
                    ┌──────────┐
                    │40 | NULL │
                    └──────────┘
```

> **Cada nodo conoce únicamente quién es el siguiente.**

#### Acceso

Para llegar al tercer elemento no existe acceso directo. No se puede saltar directamente.

Hay que recorrer:

```
10
 ↓
20
 ↓
30
```
#### Inserción

Solo cambian las referencias. No es necesario mover datos.

```
Insertar 25

Antes:
20 ─────► 30

Déspues:
20 ─────►25─────►30
```
#### Eliminación

Simplemente se modifica un enlace.

```
Eliminar 20.

Antes:
10 ─►20 ─►30

Déspues:
10 ─────────►30
```
#### Ventajas
+ Inserciones eficientes.
+ Eliminaciones eficientes.
+ No necesita memoria contigua.
+ Crece dinámicamente.

#### Desventajas
+ Acceso lento por posición.
+ Cada nodo consume memoria adicional para almacenar referencias.
+ El recorrido es menos eficiente para la caché del procesador.

#### Comparación

| Característica      | Arreglo                | Lista enlazada                                     |
| ------------------- | ---------------------- | -------------------------------------------------- |
| Memoria             | Contigua               | Dispersa                                           |
| Acceso por índice   | Muy rápido             | Lento (requiere recorrer)                          |
| Inserción al inicio | Costosa                | Muy eficiente                                      |
| Inserción al final  | Depende                | Eficiente si se mantiene referencia al último nodo |
| Inserción en medio  | Costosa                | Eficiente una vez localizado el punto              |
| Eliminación         | Costosa                | Eficiente una vez localizado el nodo               |
| Memoria extra       | No                     | Sí (referencias)                                   |
| Tamaño              | Fijo o redimensionable | Dinámico                                           |

### Idea fundamental

La diferencia esencial radica en cómo se organizan los elementos en memoria:

+ Lista secuencial (arreglo): los elementos están almacenados uno junto a otro, como una fila de casilleros numerados.

```
┌───┬───┬───┬───┐
│10 │20 │30 │40 │
└───┴───┴───┴───┘
```

+ Lista enlazada: los elementos pueden estar en cualquier lugar de la memoria; lo que los mantiene unidos son las referencias entre nodos.

```
10 ───► 20 ───► 30 ───► 40
```
#### 2.1 Listas doblemente enlazadas

¿Cómo es una lista doblemente enlazada?

Cada nodo almacena:

+ el dato
+ una referencia al nodo anterior
+ una referencia al nodo siguiente

```
┌──────────────────────────┐
│ anterior │ dato │ siguie │
└──────────────────────────┘
```
**Gracias a esto, se puede avanzar y retroceder por la lista.**
```
NULL ← [10] ⇄ [20] ⇄ [30] ⇄ [40] → NULL
```

#### ¿Qué ventajas aporta?

Al tener enlaces en ambos sentidos, algunas operaciones son más eficientes:

+ Recorrer la lista hacia adelante y hacia atrás.
+ Eliminar un nodo conocido sin tener que buscar su anterior.
+ Insertar antes o después de un nodo de forma sencilla.
+ Implementar estructuras como editores de texto, historial del navegador o listas LRU.

#### ¿Tiene algún costo?

Sí. Cada nodo necesita almacenar una referencia adicional.Esto implica:

+ mayor consumo de memoria
+ más trabajo al insertar o eliminar (hay que actualizar dos enlaces en lugar de uno)

#### ¿Cuándo se usa una u otra?

+ Lista simplemente enlazada: cuando solo necesitas recorrer en un sentido y quieres minimizar el uso de memoria.
+ Lista doblemente enlazada: cuando necesitas recorrer en ambos sentidos o realizar muchas inserciones y eliminaciones alrededor de nodos ya conocidos.

> ### La lista secuencial favorece el acceso rápido por posición, mientras que la lista enlazada favorece las modificaciones (inserciones y eliminaciones) sin mover los demás elementos.
