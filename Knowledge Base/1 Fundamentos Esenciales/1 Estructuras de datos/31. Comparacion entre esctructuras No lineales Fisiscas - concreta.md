# Comparacion entre Estructuras No Lineales Físicas

### Comparación general

| Característica     | Nodo de Árbol Binario    | Matriz de Adyacencia           | Lista de Adyacencia        |
| ------------------ | ------------------------ | ------------------------------ | -------------------------- |
| Representa         | Un nodo de un árbol      | Un grafo                       | Un grafo                   |
| Tipo de estructura | Jerárquica               | Red (grafo)                    | Red (grafo)                |
| Base física        | Nodo con dos referencias | Matriz (arreglo bidimensional) | Arreglo + listas           |
| Organización       | Padre → Hijos            | Relaciones mediante matriz     | Relaciones mediante listas |
| Conexiones máximas | 2 hijos                  | Ilimitadas según la matriz     | Ilimitadas según la lista  |
| Memoria            | O(n)                     | O(V²)                          | O(V + E)                   |

**Forma de almacenar la información**

**Nodo de Árbol Binario**

```
        dato
       /    \
izquierdo  derecho
```
Cada nodo conoce únicamente a sus dos hijos.

**Matriz de Adyacencia**
```
      A B C D

A     0 1 1 0

B     1 0 1 1

C     1 1 0 0

D     0 1 0 0
```
Cada celda indica si existe una conexión entre dos vértices.

**Lista de Adyacencia**
```
A → B → C

B → A → C → D

C → A → B

D → B
```
Solo se almacenan las conexiones existentes.

**Organización física**

**Nodo de Árbol Binario**
+ Está formado por nodos enlazados.
+ Los nodos pueden estar dispersos en memoria.

**Matriz de Adyacencia**
+ Todos los datos están almacenados de forma contigua en una matriz.

**Lista de Adyacencia**
+ Es una estructura híbrida.
+ Cada lista puede implementarse con listas enlazadas o arreglos dinámicos.

**Tipo de relaciones**

**Árbol**
+ Las relaciones son jerárquicas.
+ Cada nodo tiene un único padre (excepto la raíz).

Grafo
+ Las relaciones son arbitrarias.
+ Un vértice puede conectarse con cualquier otro.

**Número de conexiones**

**Nodo de Árbol Binario**

+ Máximo: 2 hijos - Nunca puede tener tres hijos.

**Matriz de Adyacencia y Lista de Adyacencia**

+ Un vértice puede conectarse con todos los demás.
+ Si existen `V` vértices: Hasta V−1 conexiones
+ No existe límite fijo.

**Consumo de memoria**

**Nodo de Árbol Binario**

Cada nodo almacena:

+ dato
+ referencia izquierda
+ referencia derecha

Si hay n nodos: O(n)

**Matriz de Adyacencia**

+ Siempre necesita: V × V celdas
+ Por tanto: O(V²) Aunque el grafo tenga pocas conexiones.

**Lista de Adyacencia**

Solo almacena:

+ los vértices
+ las aristas existentes.

Por tanto: O(V + E)

**Acceso a las conexiones**

**Árbol**
+ Para llegar a un nodo normalmente se sigue un camino desde la raíz.

**Matriz de Adyacencia** 
+ Consultar si existe una conexión es inmediato: M[A][D] Tiempo: O(1)

**Lista de Adyacencia**
+ Hay que recorrer la lista de vecinos: A → B → C → D Tiempo: O(grado del vértice)

**Recorrido de vecinos**

**Árbol**

Los vecinos son:

+ padre
+ hijo izquierdo
+ hijo derecho.

Normalmente los recorridos son:

+ Preorden
+ Inorden
+ Postorden
+ Por niveles

**Matriz**

Hay que revisar toda la fila.
+ 0 1 1 0 0 0 1 0 Tiempo: O(V)

**Lista**

Solo se recorren los vecinos reales.
+ A → B → C Tiempo: O(grado)

**Aplicaciones**

**Nodo de Árbol Binario**

Se utiliza para construir:

+ Árbol Binario
+ ABB
+ AVL
+ Árbol Rojo-Negro
+ Heap Binario
+ Árboles de Expresión

**Matriz de Adyacencia**

Ideal para:

+ grafos densos
+ muchas consultas de existencia de aristas
+ algoritmos donde el acceso constante a una conexión es importante.

**Lista de Adyacencia**

Ideal para:

+ redes sociales
+ mapas
+ Internet
+ dependencias entre módulos
+ sistemas de rutas
+ grafos dispersos.

**Ventajas**

| Estructura            | Ventajas                                                                      |
| --------------------- | ----------------------------------------------------------------------------- |
| Nodo de Árbol Binario | Muy flexible para representar jerarquías; base de muchos árboles eficientes.  |
| Matriz de Adyacencia  | Consulta de aristas en `O(1)`; implementación sencilla.                       |
| Lista de Adyacencia   | Menor consumo de memoria en grafos dispersos; recorrido eficiente de vecinos. |

**Desventajas**

| Estructura            | Desventajas                                                                        |
| --------------------- | ---------------------------------------------------------------------------------- |
| Nodo de Árbol Binario | No sirve para representar relaciones arbitrarias entre muchos nodos.               |
| Matriz de Adyacencia  | Consume mucha memoria (`O(V²)`); ineficiente para grafos dispersos.                |
| Lista de Adyacencia   | Verificar la existencia de una arista puede requerir recorrer la lista de vecinos. |


**Relación entre ellas**

> Aunque las tres pertenecen a las estructuras concretas no lineales, representan conceptos distintos:

```
Estructuras No Lineales

├── Árboles
│
│   Nodo de Árbol Binario
│        │
│        ├── Árbol Binario
│        ├── ABB
│        ├── AVL
│        ├── Rojo-Negro
│        └── Heap
│
└── Grafos
    │
    ├── Matriz de Adyacencia
    └── Lista de Adyacencia
```

+ El Nodo de Árbol Binario es el componente físico con el que se construyen los árboles.
+ La Matriz de Adyacencia y la Lista de Adyacencia son dos formas alternativas de representar el mismo TDA Grafo.

### Resumen

| Aspecto                           | Nodo de Árbol Binario       | Matriz de Adyacencia | Lista de Adyacencia |
| --------------------------------- | --------------------------- | -------------------- | ------------------- |
| Implementa                        | Árboles                     | Grafos               | Grafos              |
| Organización                      | Jerárquica                  | Red                  | Red                 |
| Estructura base                   | Nodo con 2 referencias      | Matriz (`V × V`)     | Arreglo + listas    |
| Memoria                           | O(n)                        | O(V²)                | O(V + E)            |
| Consultar una conexión específica | No es su objetivo principal | **O(1)**             | O(grado)            |
| Recorrer conexiones de un nodo    | O(1) a hijos directos       | O(V)                 | **O(grado)**        |
| Mejor escenario                   | Datos jerárquicos           | Grafos densos        | Grafos dispersos    |

### Ideas clave

Las tres estructuras pertenecen a las estructuras concretas no lineales, pero cumplen propósitos diferentes:

+ El Nodo de Árbol Binario modela jerarquías, donde cada nodo tiene como máximo dos hijos.
+ La Matriz de Adyacencia modela grafos mediante una matriz, ofreciendo acceso muy rápido para comprobar si existe una conexión entre dos vértices.
+ La Lista de Adyacencia también modela grafos, pero almacena únicamente las conexiones existentes, siendo mucho más eficiente en memoria para grafos dispersos.

árboles y grafos son dos familias distintas de estructuras no lineales: el primero organiza la información de forma jerárquica, mientras que el segundo permite representar relaciones arbitrarias entre elementos.
