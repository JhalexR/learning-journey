# ¿Qué es un Grafo (Graph)?

Un Grafo es un Tipo de Dato Abstracto (TDA) que representa un conjunto de vértices (nodos) conectados mediante aristas (edges).

Su característica principal es:

+ Modela relaciones entre elementos, sin imponer una estructura jerárquica o lineal.

**Analogía**

Una red social.

```
Ana ----- Juan
 |          |
 |          |
Pedro ----- Luis
```
+ Cada persona es un vértice.
+ Cada amistad es una arista.

**No existe un "inicio" o una "raíz"; cualquier nodo puede conectarse con varios otros.**

### Componentes de un grafo

**Vértices**

Representan los objetos.

+ A   B   C   D

**Aristas**

Representan las relaciones.

+ A ----- B

**Grafo completo**

```
      A
     / \
    B---C
     \
      D
```

**Conjunto de vértices**

+ V = {A,B,C,D}

**Conjunto de aristas**

+ E = { (A,B), (A,C), (B,C), (B,D) }

### ¿En qué se diferencia de un árbol?

Un árbol es un caso particular de grafo.

| Árbol                                   | Grafo                       |
| --------------------------------------- | --------------------------- |
| Tiene una raíz.                         | No necesariamente.          |
| No tiene ciclos.                        | Puede tener ciclos.         |
| Existe un único camino entre dos nodos. | Puede haber varios caminos. |
| Es jerárquico.                          | Es una red de relaciones.   |

**Ejemplo de árbol:**

```
      A
     / \
    B   C
       /
      D
```

**Ejemplo de grafo:**
```
A ----- B
|       |
|       |
C ----- D

Aquí existen varios caminos entre A y D.
```

## Tipos de grafos

### 1. Grafo no dirigido

Las conexiones no tienen dirección.

```
A ----- B

Significa:

A está conectado con B

B está conectado con A
```
<br>
Ejemplos:

+ amistades
+ carreteras de doble sentido

### 2. Grafo dirigido (Digraph)

Las aristas tienen dirección.

+ A -----> B -> No implica que B pueda volver a A.

<br>
Ejemplos:

+ seguidores en redes sociales
+ dependencias entre tareas
+ llamadas entre funciones

### Grafos ponderados

Las aristas pueden tener un peso.

```
      10
A ------------ B
 \            /
5 \          /3
   \        /
      C
```
</br>
Los números pueden representar:

+ distancia
+ tiempo
+ costo
+ capacidad

<br>
Ejemplos:

+ GPS
+ rutas aéreas
+ redes eléctricas

### Grafos no ponderados

Todas las conexiones valen lo mismo.

```
A ---- B
 \   /
   C

   Solo importa si existe o no la conexión.
```

### Operaciones básicas

| Operación        | Descripción                                |
| ---------------- | ------------------------------------------ |
| Crear            | Construir un grafo vacío.                  |
| Agregar vértice  | Insertar un nuevo nodo.                    |
| Eliminar vértice | Quitar un nodo.                            |
| Agregar arista   | Conectar dos nodos.                        |
| Eliminar arista  | Eliminar una conexión.                     |
| Son adyacentes   | Verificar si dos nodos están conectados.   |
| Obtener vecinos  | Recuperar los nodos conectados a uno dado. |
| Recorrer         | Visitar todos los vértices.                |

## Implementaciones

El TDA Grafo puede implementarse principalmente de dos formas

### 1. Matriz de adyacencia

```
      A B C D

A     0 1 1 0

B     1 0 1 1

C     1 1 0 0

D     0 1 0 0
```
**Un 1 indica que existe conexión.**

<br>
Ventajas
+ Muy rápida para comprobar si existe una arista (O(1)).
+ Sencilla de implementar.

Desventajas
+ Consume mucha memoria cuando el grafo tiene pocas conexiones.

### 2. Lista de adyacencia

```
A → B → C

B → A → C → D

C → A → B

D → B
```
```
A → B → C

B → A → C → D

C → A → B

D → B
```
Cada vértice almacena solo sus vecinos.

<br>
Ventajas
+ Mucho menor consumo de memoria para grafos dispersos.
+ Muy utilizada en la práctica.

<br>
Desventajas
+ Comprobar si dos nodos están conectados requiere recorrer la lista de vecinos.

### Recorridos

Los dos algoritmos fundamentales son:

**DFS (Depth-First Search)**

Explora un camino hasta el final antes de retroceder.

+ A -> B -> D <--> C

> Suele implementarse con una pila (explícita o mediante la pila de llamadas de la recursión).**

**BFS (Breadth-First Search)**

Explora todos los vecinos antes de pasar al siguiente nivel.

```
Nivel 0

A

↓

Nivel 1

B   C

↓

Nivel 2

D   E
```
> Suele implementarse con una cola.

**Complejidad Temporal**

| Operación        | Matriz |         Lista        |
| ---------------- | :----: | :------------------: |
| Agregar arista   |  O(1)  |         O(1)         |
| Verificar arista |  O(1)  | O(grado del vértice) |
| Recorrer vecinos |  O(n)  | O(grado del vértice) |
| Memoria          |  O(V²) |       O(V + E)       |

<br>
Donde:

+ V = número de vértices.
+ E = número de aristas.

### Aplicaciones

Los grafos aparecen en muchos problemas reales:

+ redes sociales
+ mapas y GPS
+ redes de computadoras
+ rutas aéreas
+ dependencias entre módulos de software
+ motores de recomendación
+ análisis de redes eléctricas
+ planificación de proyectos

**Comparación con otras estructuras**

| Estructura | Organización      |
| ---------- | ----------------- |
| Lista      | Secuencial        |
| Pila       | LIFO              |
| Cola       | FIFO              |
| Árbol      | Jerárquica        |
| Grafo      | Red de relaciones |

**Relación entre Árbol y Grafo**

```
Grafo
│
├── Puede tener ciclos
├── Puede estar desconectado
├── Puede tener múltiples caminos
│
└── Árbol
     ├── Siempre conectado
     ├── Sin ciclos
     ├── Un único camino entre dos nodos
     └── Tiene n−1 aristas para n nodos
```
> Esto significa que todo árbol es un grafo, pero no todo grafo es un árbol.

### Idea clave

+ El TDA Grafo es la estructura más general para representar relaciones entre elementos
+ Un grafo permite que cualquier nodo pueda relacionarse con cualquier otro, formando redes complejas.
+ Es la estructura ideal para modelar problemas donde importan las conexiones, los caminos y las relaciones
