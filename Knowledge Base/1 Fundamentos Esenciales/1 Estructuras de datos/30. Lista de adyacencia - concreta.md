# ¿Qué es una Lista de Adyacencia?

Una Lista de Adyacencia representa un grafo mediante una colección donde cada vértice mantiene una lista de sus vecinos.

es la más utilizada para representar grafos dispersos (con pocas conexiones)

Supongamos:

```
      A
     / \
    B---C
     \
      D
```

Las conexiones son:

```
A-B

A-C

B-C

B-D
```

La Lista de Adyacencia será:

```
A → B → C

B → A → C → D

C → A → B

D → B
```

**almacena solo las conexiones que realmente existen.**

**Comparación visual con la Matriz**

**Matriz de Adyacencia**

|       |  A |  B |  C |  D |
| ----- | -: | -: | -: | -: |
| **A** |  0 |  1 |  1 |  0 |
| **B** |  1 |  0 |  1 |  1 |
| **C** |  1 |  1 |  0 |  0 |
| **D** |  0 |  1 |  0 |  0 |

Hay muchas celdas con: 0

**Lista de Adyacencia**

```
A → B → C

B → A → C → D

C → A → B

D → B
```

**Aquí no existe información redundante sobre las conexiones inexistentes.**

### ¿Cómo se implementa físicamente?

Normalmente mediante:

+ un arreglo (o arreglo dinámico) donde cada posición representa un vértice
+ y cada posición contiene una lista enlazada o un arreglo dinámico con sus vecinos.

```
Índice

0 → Lista de vecinos

1 → Lista de vecinos

2 → Lista de vecinos

3 → Lista de vecinos
```

**Grafo dirigido** 

Supongamos:

```
A → B

A → C

C → D
```

La Lista de Adyacencia será:

```
A → B → C

B

C → D

D
```
+ B no apunta a A
+ D no apunta a C.

**Solo almacenamos las aristas que realmente existen.**

**Grafo no dirigido**

```
A ----- B

A ----- C

B ----- D
```

La representación será:
```
A → B → C

B → A → D

C → A

D → B
```
Cada arista aparece dos veces:
```
A → B

B → A
```
**porque la conexión existe en ambos sentidos.**

**Grafos ponderados**

```
A --5-- B

A --2-- C

B --8-- C
```

La lista puede ser:
```
A → (B,5) → (C,2)

B → (A,5) → (C,8)

C → (A,2) → (B,8)
```

### Operaciones

**Agregar una arista**
```
A → B → C

Solo añadimos D a la lista de A.

A → B → C → D

Tiempo: O(1)
```

**Eliminar una arista**

```
A → B → C → D

Queremos eliminar:

A → C

Debemos recorrer la lista de A hasta encontrar C.

A → B → D

Tiempo: O(grado(A))
```

**Obtener los vecinos**

**Esta es la gran ventaja.**

Para conocer los vecinos de: A

Simplemente recorremos:

A → B → C

No necesitamos revisar conexiones inexistentes.

Tiempo: O(grado(A))

donde el grado es el número de vecinos del vértice.

**Verificar si existe una arista**

¿Existe A → C?

Debemos buscar C dentro de la lista de A.

Tiempo: O(grado(A)) 

*Aquí la Matriz de Adyacencia es más rápida (O(1)).*

### Complejidad Temporal

Si:

+ `V` = número de vértices.
+ `E` = número de aristas.

| Operación              | Complejidad |
| ---------------------- | ----------: |
| Agregar arista         |       O(1)* |
| Eliminar arista        |    O(grado) |
| Verificar arista       |    O(grado) |
| Obtener vecinos        |    O(grado) |
| Recorrer todo el grafo |    O(V + E) |

### Consumo de memoria

**Aquí aparece su mayor ventaja.**

Solo almacenamos las aristas existentes.

+ La memoria utilizada es: O(V + E)
+ En cambio, la Matriz de Adyacencia siempre necesita: O(V²) aunque existan pocas conexiones.

**Ejemplo de grafo disperso**

Supongamos

A──B

C──D

E

La Matriz sería:

|       |  A |  B |  C |  D |  E |
| ----- | -: | -: | -: | -: | -: |
| **A** |  0 |  1 |  0 |  0 |  0 |
| **B** |  1 |  0 |  0 |  0 |  0 |
| **C** |  0 |  0 |  0 |  1 |  0 |
| **D** |  0 |  0 |  1 |  0 |  0 |
| **E** |  0 |  0 |  0 |  0 |  0 |

**La mayoría de las celdas son ceros.**

La Lista de Adyacencia sería:

A → B

B → A

C → D

D → C

E

**Mucho más compacta.**

**Ejemplo de grafo denso**

Ahora imaginemos:

```
A conectado con todos

B conectado con todos

C conectado con todos

D conectado con todos
```

La Lista de Adyacencia sería:

```
A → B → C → D

B → A → C → D

C → A → B → D

D → A → B → C
```

Aquí casi termina almacenando una cantidad de información similar a la Matriz.

> Por eso la Matriz suele ser preferible para grafos muy densos.

**Comparación con la Matriz de Adyacencia**

| Característica       | Matriz de Adyacencia | Lista de Adyacencia |
| -------------------- | -------------------- | ------------------- |
| Estructura base      | Matriz (`n × n`)     | Arreglo de listas   |
| Memoria              | O(V²)                | O(V + E)            |
| Verificar una arista | O(1)                 | O(grado)            |
| Obtener vecinos      | O(V)                 | O(grado)            |
| Ideal para           | Grafos densos        | Grafos dispersos    |
| Agregar vértices     | Costoso              | Más sencillo        |
```
TDA Grafo

│

├── Matriz de Adyacencia
│     └── Basada en una Matriz
│
└── Lista de Adyacencia
      └── Basada en un Arreglo + Listas
```
### ¿Cuál debo usar?

Usa una Matriz de Adyacencia cuando:

+ el grafo es muy denso
+ consultas constantemente si existe una arista
+ el número de vértices es relativamente pequeño

Usa una Lista de Adyacencia cuando:

+ el grafo es disperso
+ necesitas recorrer frecuentemente los vecinos de un vértice
+ la memoria es importante
+ el grafo puede crecer dinámicamente

> En la mayoría de aplicaciones reales (redes sociales, mapas, rutas, dependencias entre módulos, etc.), los grafos son dispersos, por lo que la Lista de Adyacencia suele ser la representación preferida.

### Relación con las estructuras concretas

_**Una Lista de Adyacencia no es una estructura completamente nueva, sino una combinación de estructuras**_

```
Lista de Adyacencia

│

├── Arreglo (para almacenar los vértices)

└── Lista Enlazada o Arreglo Dinámico
    (para almacenar los vecinos)
```

### Ideas clave

+ La Lista de Adyacencia es una estructura concreta para representar grafos donde cada vértice almacena únicamente la lista de sus vecinos
+ Su principal ventaja es que solo almacena las conexiones que realmente existen
+ Esta representación es la que encontrarás con mayor frecuencia en algoritmos clásicos sobre grafos
