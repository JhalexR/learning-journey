# ¿Qué es un Diccionario / Mapa?

Un Diccionario (Dictionary) o Mapa (Map) es un Tipo de Dato Abstracto (TDA) que almacena una colección de pares (clave, valor).

La idea fundamental es:

_**Cada clave identifica exactamente un valor.**_

> El Diccionario (también llamado Mapa o Map) es probablemente el TDA asociativo más importante.

#### Analogía

una agenda telefónica.

```
Nombre  →  Teléfono

Juan    →  3101234567
Ana     →  3209876543
Pedro   →  3154567890
```

Aquí:

+ Nombre = clave.
+ Teléfono = valor.

Cuando buscas "Ana", obtienes su número.

Un mapa puede representarse como:

| Diccionario/mapa | Claves   | Valores |
| ---------------- | :------: | :-----: |
| "Juan"  → 25     |   Juan   |   25    |
| "Ana"   → 31     |   Ana    |   31    |
| "Pedro" → 28     |   Pedro  |   28    |

**Cada clave aparece una sola vez.**

#### Características principales

Un mapa:

+ almacena pares clave–valor
+ no permite claves duplicadas
+ normalmente sí permite valores duplicados
+ permite buscar un valor mediante su clave
+ insertar una clave existente suele actualizar su valor

#### Operaciones básicas

| Operación        | Descripción                               |
| ---------------- | ----------------------------------------- |
| Crear            | Construir un mapa vacío.                  |
| Put(k,v)         | Insertar o actualizar un par clave–valor. |
| Get(k)           | Obtener el valor asociado a una clave.    |
| Remove(k)        | Eliminar una clave y su valor.            |
| ContainsKey(k)   | Verificar si una clave existe.            |
| ContainsValue(v) | Verificar si un valor existe (opcional).  |
| Keys()           | Obtener todas las claves.                 |
| Values()         | Obtener todos los valores.                |
| Entries()        | Obtener todos los pares clave–valor.      |
| Size()           | Número de pares almacenados.              |
| isEmpty()        | Verificar si está vacío.                  |

## Implementaciones

### 1. Tabla hash (Hash Map)

Es la implementación más utilizada.

```Hash("Ana")
       │
       ▼

┌────┬────┬────┬────┐
│    │Ana │    │Juan│
└────┴────┴────┴────┘
```
#### Complejidad promedio:

| Operación | Tiempo |
| --------- | ------ |
| Put       | O(1)   |
| Get       | O(1)   |
| Remove    | O(1)   |

> No mantiene las claves ordenadas.

### 2. Árbol de búsqueda balanceado

```
        Luis
       /    \
    Ana     Pedro
      \        \
     Juan     Sofía
```
#### Complejidad promedio:
| Operación | Tiempo   |
| --------- | -------- |
| Put       | O(log n) |
| Get       | O(log n) |
| Remove    | O(log n) |

> Ventaja: permite recorrer las claves en orden.

### 3. Lista o arreglo

También es posible implementar un mapa con una lista de pares:

```
[
 (Juan,25),
 (Ana,31),
 (Pedro,28)
]
```
Para buscar "Pedro" hay que recorrer la lista hasta encontrar la clave.
#### Complejidad promedio:
| Operación | Tiempo |
| --------- | ------ |
| Get       | O(n)   |
| Put       | O(n)   |
| Remove    | O(n)   |

> Se usa principalmente con fines didácticos o cuando el número de elementos es pequeño.

#### Aplicaciones

Los mapas aparecen en casi cualquier sistema de software:

+ agenda de contactos (nombre → teléfono)
+ inventarios (código → producto)
+ usuarios (id → objeto Usuario)
+ cachés (URL → contenido)
+ configuraciones (clave → valor)
+ diccionarios lingüísticos (palabra → definición)

#### Idea clave

**Permite acceder rápidamente a información mediante un identificador único, combinando búsqueda eficiente con asociación de datos.**
