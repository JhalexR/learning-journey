# Comparacion asociativos y de búsqueda TDA

Diferencias entres Conjuto(set), diccionario/mapa, Cola de prioridad

| Característica           | Conjunto (Set)   | Diccionario / Mapa (Map)             | Cola de Prioridad (Priority Queue) |
| ------------------------ | ---------------- | ------------------------------------ | ---------------------------------- |
| Almacena                 | Elementos únicos | Pares clave–valor                    | Elementos con prioridad            |
| Duplicados               | No               | No en las claves (sí en los valores) | Depende de la implementación       |
| Acceso principal         | Por pertenencia  | Por clave                            | Por prioridad                      |
| Orden natural            | Generalmente no  | Depende de la implementación         | Siempre según la prioridad         |
| Operación más importante | `Contains()`     | `Get(clave)`                         | `Extract()`                        |

Aunque las tres pertenecen a la categoría de TDA asociativos y de búsqueda, cada una resuelve un problema diferente. La diferencia principal está en qué almacenan y cómo se accede a los elementos.

### 1. Conjunto (Set)

**¿Qué responde?**
+ ¿Este elemento pertenece al conjunto?

**Su objetivo principal es:**
+ evitar duplicados
+ verificar pertenencia rápidamente

> No interesa asociar información adicional a cada elemento.

### 2. Diccionario / Mapa (Map)

**¿Qué responde?**
+ ¿Qué valor corresponde a esta clave?

**Su objetivo principal es:**
+ No solo interesa saber si un elemento existe, sino recuperar la información asociada a ese elemento.

### 3. Cola de Prioridad

**¿Qué responde?**
+ ¿Cuál es el elemento más importante en este momento?

**Su objetivo principal es:**
+ No importa cuándo llegó D.
+ Solo importa que posee la mayor prioridad.

| Set (solo elementos)| Map (valor asociado)| Priority Queue |

|           A         |         A → 25      |      A (2)     |
|           B         |         B → 31      |      B (1)     |
|           C         |         C → 28      |      C (3)     |

#### Forma de acceso

Esta es probablemente la diferencia más importante.

**Set**
+ ¿Existe este elemento?

**Map**
+ ¿Qué valor tiene esta clave?

**Priority Queue**
+ Priority Queue

#### Operaciones principales

| TDA            | Operación más utilizada     |
| -------------- | --------------------------- |
| Set            | Add(), Remove(), Contains() |
| Map            | Put(), Get(), Remove()      |
| Priority Queue | Insert(), Peek(), Extract() |

#### Implementaciones típicas

| TDA            | Implementación común         |
| -------------- | ---------------------------- |
| Set            | Tabla hash, árbol balanceado |
| Map            | Tabla hash, árbol balanceado |
| Priority Queue | Heap (montículo)             |

> Set y Map suelen compartir implementaciones, mientras que la Priority Queue casi siempre utiliza un heap, porque necesita mantener el elemento de mayor o menor prioridad accesible.

### Ejemplo casos de uso

**Set**
+ Eliminar duplicados.

**Map**
+ Asociar datos.

**Priority Queue**
+ Atender tareas urgentes.

Imagina un comercio que vende productos a traves de internet.

_Set_ 

1. El sitio web de ventas del comercio pide crear una cuenta donde debes registrar tus datos personales.
2. Te registras utilizando tu correo email@email.com pero ese dia decides no comprar.
3. Meses despues ves algo que te gusta en el mismo comercio y quieres registrarte otra vez.
4. Olvidates que ya tenias una cuenta y al intenar registrate nuevamente con el correo email@email.com el sitio web te informa que ya tienes una cuenta creada.

_en esta parte hipoteticamente, se pudo haber usado la estructura **set** para almacenar los usaurios para evitar que se registre dos veces el mismo email_

_Map_ 

5. Realizas la compra de un producto para envio a tu domicilio.
6. La empresa de logistica consulta tu correo email@email.com para conocer tu dirección asociada a tu correo

_en esta parte hipoteticamente, se pudo haber usado la estructura  **Map** para evitar que se registre dos veces el mismo usuario_

_Priority Queue_

7. Cuando hiciste la compra escogiste la opcion de _"Envío Express"_ la cual consiste en que por un valor adicional te entregan el mismo dia.
8. En la informacion del pedido el comercio lo etiqueta con prioridad de entrega (1) antes de los otros pedidos que no tienen la opcion de _"Envío Express"_

_en esta parte hipoteticamente, se pudo haber usado la estructura  **Priority Queue** para entregar tu pedido primero, incluso antes que pedidos que se hicieron antes que el tuyo_

**Resumen conceptual**

| TDA                | Pregunta principal                       |
| ------------------ | ---------------------------------------- |
| **Set**            | ¿El elemento existe?                     |
| **Map**            | ¿Qué valor corresponde a esta clave?     |
| **Priority Queue** | ¿Cuál es el elemento de mayor prioridad? |

### Idea clave

Aunque los tres pertenecen a la familia de TDA asociativos y de búsqueda, cada uno está diseñado para un objetivo distinto:

+ Conjunto (Set): gestionar elementos únicos y responder consultas de pertenencia.
+ Diccionario / Mapa (Map): asociar claves únicas con valores y recuperar información de forma eficiente.
+ Cola de Prioridad (Priority Queue): organizar elementos según una prioridad, de modo que siempre pueda obtenerse primero el más importante, sin importar el orden de inserción.
