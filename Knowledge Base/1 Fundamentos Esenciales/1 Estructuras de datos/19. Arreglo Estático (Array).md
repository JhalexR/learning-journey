# ¿Qué es un Arreglo Estático (Array)?

Un Arreglo Estático (Array) es una estructura de datos concreta que almacena una colección de elementos del mismo tipo en posiciones contiguas de memoria y cuyo tamaño se define al momento de crearlo y no puede cambiar.

Sus características principales son:

+ Todos los elementos son del mismo tipo.
+ Se almacenan en memoria contigua.
+ Cada elemento tiene un índice.
+ El tamaño es fijo.

**Analogía**

una fila de casilleros numerados.

```
Índice

0    1    2    3    4

┌───┬───┬───┬───┬───┐
│10 │20 │30 │40 │50 │
└───┴───┴───┴───┴───┘
```
<br>
Cada casillero tiene:

+ una posición (índice)
+ un contenido.

### Memoria

Lo que hace especial a un array es que todos los elementos ocupan posiciones consecutivas en memoria.

```
Dirección

1000
1004
1008
1012
1016

Si cada entero ocupa 4 bytes:

1000 → 10

1004 → 20

1008 → 30

1012 → 40

1016 → 50

No existen huecos entre ellos.
```

**Acceso por índice**

Gracias a que la memoria es contigua, acceder a cualquier elemento es inmediato.

+ Por ejemplo: array[3] Resultado: 40

El procesador calcula la dirección mediante una fórmula:

+ Dirección = DirecciónBase + Índice × TamañoElemento

Ejemplo:

```
Base = 1000

Índice = 3

Elemento = 4 bytes

↓

1000 + 3×4

↓

1012
```
> No necesita recorrer los elementos anteriores.

### Operaciones básicas

**Leer**

+ array[2] - Resultado: 30 - O(1)

**Modificar**

+ array[2] = 35 - 10 20 35 40 50 - O(1)

**Recorrer**

+ 10, 20, 30, 40, 50 - Tiempo: O(n) Porque hay que visitar todos los elementos.

**Buscar**

+ Si el arreglo no está ordenado: Tiempo: O(n) 
+ Si está ordenado, puede utilizarse búsqueda binaria, reduciendo el tiempo a: O(log n)

**Inserción**

Supongamos: 10 20 30 40

Queremos insertar 25 en la posición 2.

Antes:
```
Índice

0 1 2 3

10 20 30 40
```
Se deben mover los elementos hacia la derecha.
```
10 20 25 30 40
```
Esto implica copiar varios elementos - Tiempo: O(n)

**Eliminación**

Eliminar el 20.

+ Antes: 10 20 30 40

+ Después: 10 30 40 -> Ahora hay que mover los elementos restantes hacia la izquierda.

+ Tiempo: O(n)

### Tamaño fijo

Esta es la característica que define al arreglo estático.

Si se crea: 
```C++
int datos[5]
```
Solo existen cinco posiciones.
```
┌───┬───┬───┬───┬───┐
│   │   │   │   │   │
└───┴───┴───┴───┴───┘
```
**No puede convertirse automáticamente en un arreglo de diez posiciones.**

<br>
Si se necesita más espacio, normalmente hay que:

+ crear un arreglo nuevo
+ copiar todos los elementos
+ liberar el anterior.

> Los arreglos dinámicos automatizan este proceso, pero el arreglo estático no.

### Complejidad temporal

| Operación                            |  Tiempo  |
| ------------------------------------ | :------: |
| Acceder por índice                   |   O(1)   |
| Modificar                            |   O(1)   |
| Recorrer                             |   O(n)   |
| Buscar (desordenado)                 |   O(n)   |
| Buscar (ordenado + búsqueda binaria) | O(log n) |
| Insertar                             |   O(n)   |
| Eliminar                             |   O(n)   |

**Ventajas**
+ Acceso aleatorio muy rápido O(1).
+ Estructura simple.
+ Excelente uso de la memoria caché del procesador.
+ Bajo consumo de memoria adicional.
+ Ideal cuando el tamaño de los datos es conocido.


**Desventajas**
+ Tamaño fijo.
+ Inserciones costosas.
+ Eliminaciones costosas.
+ Necesita memoria contigua.


**Aplicaciones**

Los arreglos son la base de muchas estructuras y algoritmos:

+ matrices
+ imágenes (cada píxel ocupa una posición)
+ tablas
+ vectores
+ implementaciones de listas secuenciales;¡
+ pilas basadas en arreglos
+ colas circulares
+ heaps (montículos)

### Se utiliza principalmente para implementar
>dependiendo de si se requiere que el tamaño máximo se conozca de antemano.

Es la estructura base para implementar TDAs con comportamientos específicos como:
+ Lista Secuencial o Vector
+ Pilas (Stacks) 
+ Colas (Queues)

### Ideas clave

El Arreglo Estático (Array) es la estructura concreta más simple y eficiente para almacenar datos cuando:

+ el número de elementos es conocido de antemano
+ se necesita acceder rápidamente por índice
+ las inserciones y eliminaciones son poco frecuentes.

Su gran fortaleza es el acceso aleatorio en tiempo constante (O(1)), posible gracias a que los elementos se almacenan de forma contigua en memoria. Su principal limitación es que su tamaño no puede crecer ni disminuir una vez creado,
