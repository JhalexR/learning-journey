# ¿Qué es un Conjunto (Set)?

Un Conjunto (Set) es un Tipo de Dato Abstracto (TDA) que almacena una colección de elementos únicos, sin permitir duplicados.

#### Características principales

Su característica principal es:
+ Un elemento puede pertenecer al conjunto una sola vez

Un conjunto:

+ no permite elementos repetidos;
+ normalmente no tiene un orden significativo (aunque algunas implementaciones sí lo conservan);
+ permite comprobar rápidamente si un elemento pertenece al conjunto
+ cada elemento aparece como máximo una vez.

Ejemplo 1:

```
{2, 5, 8, 10}
es un conjunto válido.

es un conjunto válido.
no lo es, porque el 5 aparece dos veces.
```
Ejemplo 2:
```
A = {5,8}

Add(5)

A = {5,8}
```
No ocurre ningún cambio porque el elemento ya existe.

#### Operaciones básicas

| Operación      | Descripción                                     |
| -------------- | ----------------------------------------------- |
| Crear          | Construir un conjunto vacío.                    |
| Add(x)         | Agregar un elemento.                            |
| Remove(x)      | Eliminar un elemento.                           |
| Contains(x)    | Verificar si un elemento pertenece al conjunto. |
| Size()         | Número de elementos.                            |
| isEmpty()      | Verificar si está vacío.                        |
| Clear()        | Eliminar todos los elementos.                   |
| Union()        | Unión de conjuntos.                             |
| Intersection() | Intersección de conjuntos.                      |
| Difference()   | Diferencia entre conjuntos.                     |

El TDA Set está inspirado directamente en la teoría de conjuntos.

Si tenemos:
```
A = {1,2,3}

B = {3,4,5}
```` 
podemos realizar operaciones como:
```
Unión
A ∪ B = {1,2,3,4,5}

Intersección
A ∩ B = {3}

Diferencia
A − B = {1,2}

Diferencia simétrica
A △ B = {1,2,4,5}
```
Muchos lenguajes implementan estas operaciones de forma directa.

### Implementaciones

#### 1. Tabla hash (Hash Set)

Es la implementación más común.
```
Hash(x)
     │
     ▼
┌───┬───┬───┬───┬───┐
│   │ 8 │   │15 │22 │
└───┴───┴───┴───┴───┘
```
Ventajas:

+ búsqueda muy rápida
+ inserción rápida
+ eliminación rápida

Complejidad promedio:

| Operación | Tiempo |
| --------- | ------ |
| Add       | O(1)   |
| Remove    | O(1)   |
| Contains  | O(1)   |

#### 2. Árbol de búsqueda balanceado

```
        20
       /  \
     10    35
    / \    / \
   5 15 30 40
```
Ventajas:

+ mantiene los elementos ordenados
+ permite recorrerlos de menor a mayor

Complejidad: 

| Operación | Tiempo   |
| --------- | -------- |
| Add       | O(log n) |
| Remove    | O(log n) |
| Contains  | O(log n) |

#### 3. Lista o arreglo

También es posible implementar un conjunto mediante una lista.

Sin embargo, antes de insertar hay que verificar que el elemento no exista.

```
[5,8,12]

Agregar 8

↓

Recorrer toda la lista

↓

No insertar
```
> Esta implementación suele tener costo O(n) para búsqueda e inserción, por lo que solo es adecuada para conjuntos pequeños o con fines didácticos.

#### Comparaciond de implementaciones concretas

| Implementación   | Búsqueda | Inserción | Orden |
| ---------------- | :------: | :-------: | :---: |
| Lista            |   O(n)   |    O(n)   |   Sí  |
| Arreglo          |   O(n)   |    O(n)   |   Sí  |
| Hash Set         |   O(1)&  |   O(1)&   |   No  |
| Árbol balanceado | O(log n) |  O(log n) |   Sí  |

> & En promedio.

#### Aplicaciones

Los conjuntos son útiles cuando interesa saber si un elemento ya existe, sin permitir duplicados:

+ usuarios registrados;
+ palabras únicas en un texto;
+ etiquetas (tags);
+ permisos de un usuario;
+ visitantes únicos de un sitio web;
+ eliminar duplicados de una colección.

#### Comparación con la Lista

| Lista                                         | Conjunto                            |
| --------------------------------------------- | ----------------------------------- |
| Permite duplicados.                           | No permite duplicados.              |
| El orden suele ser importante.                | El orden generalmente no importa.   |
| Se accede por posición.                       | Se consulta por pertenencia.        |
| Puede haber varias copias del mismo elemento. | Cada elemento aparece una sola vez. |

Mientras que un TDA Lista responde preguntas como:

+ ¿Qué elemento está en la posición 5?
+ ¿Cuál es el siguiente elemento?

un TDA Set responde preguntas como:

+ ¿El elemento pertenece al conjunto?
+ ¿Ya existe este elemento?
+ ¿Qué elementos son comunes entre dos conjuntos?

#### Idea clave

**_El Conjunto cambia el enfoque desde el orden hacia la unicidad y la pertenencia, lo que lo convierte en una de las estructuras más utilizadas para búsquedas rápidas y eliminación de duplicados._**
