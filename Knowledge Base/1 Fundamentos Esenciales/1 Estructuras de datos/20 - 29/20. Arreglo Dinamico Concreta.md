# ¿Qué es un Arreglo Dinámico?

Un Arreglo Dinámico (Dynamic Array) es una estructura de datos concreta que almacena elementos en memoria contigua, igual que un arreglo estático, pero que puede aumentar (y en algunas implementaciones disminuir) su capacidad automáticamente cuando es necesario.

**Se comporta como un arreglo normal, pero administra internamente el crecimiento de la memoria.**

Es la evolución natural del Arreglo Estático. Conserva casi todas sus ventajas (acceso rápido por índice y memoria contigua), pero elimina su principal limitación: el tamaño fijo.

**Es una de las estructuras concretas más utilizadas y es la base de clases como:**

| Lenguaje   | Implementación                                                                          |
| ---------- | --------------------------------------------------------------------------------------- |
| Java       | `ArrayList<E>`                                                                          |
| C++        | `std::vector<T>`                                                                        |
| C#         | `List<T>`                                                                               |
| Python     | `list` (implementado como un arreglo dinámico con una estrategia de crecimiento propia) |
| JavaScript | `Array` (internamente optimizado por el motor del lenguaje)                             |


**Estructura interna**

Generalmente mantiene tres datos:
+ Datos
+ Capacidad (Capacity)
+ Tamaño (Size)

Ejemplo:
```
Índice

0 1 2 3 4 5 6 7

┌──┬──┬──┬──┬──┬──┬──┬──┐
│2 │8 │5 │9 │1 │  │  │  │
└──┴──┴──┴──┴──┴──┴──┴──┘

Size = 5
Capacity = 8

Los espacios vacíos están reservados para futuras inserciones.
```

### ¿Qué ocurre cuando se llena?

Supongamos:

```
┌──┬──┬──┬──┐
│1 │2 │3 │4 │
└──┴──┴──┴──┘

Ahora agregamos: 5 

Add(5)
```

**No existe espacio, entonces ocurre el siguiente proceso.**

**Paso 1**

Crear un arreglo más grande, muchas implementaciones duplican la capacidad.

+ Capacity 4 -> 8

**Paso 2**

Copiar los elementos.

```
Viejo

1 2 3 4

↓

Nuevo

1 2 3 4 _ _ _ _
```
**Paso 3**

Insertar el nuevo elemento.

+ 1 2 3 4 5 _ _ _

**Paso 4**

Liberar el arreglo antiguo.

+ El usuario no nota nada.

### ¿Por qué duplicar el tamaño?

Supongamos que aumentáramos de uno en uno:
+ Cada inserción requeriría copiar todos los elementos
+ Eso sería muy costoso.

Duplicando la capacidad:
+ Las copias son poco frecuentes
+ lo que hace que las inserciones al final tengan un costo amortizado de O(1).

### Operaciones básicas

**Acceder**

+ array[3] Tiempo = O(1)

**Modificar**

array[2] = 10 Tiempo = O(1)

**Agregar al final**

Normalmente Tiempo = O(1), Cuando hay redimensionamiento Tiempo = O(n)

**Insertar en una posición**

```
10 20 30 40

↓

Insertar 25

↓

10 20 25 30 40
```
+ Tiempo = O(n) -> Hay que desplazar elementos.

**Eliminar**

```
10 20 30 40

↓

Eliminar 20

↓

10 30 40
```
+ Tiempo = O(n) -> Hay que desplazar elementos.

### Complejidad Temporal

| Operación                |      Tiempo     |
| ------------------------ | :-------------: |
| Acceder por índice       |       O(1)      |
| Modificar                |       O(1)      |
| Agregar al final         | O(1) amortizado |
| Insertar en una posición |       O(n)      |
| Eliminar                 |       O(n)      |
| Buscar                   |       O(n)      |

> ¿Qué significa "amortizado"?
>> Solo unas pocas inserciones requieren copiar todos los elementos, La mayoría simplemente ocupan un espacio libre.
>>> Si se reparte el costo de esa copia entre todas las inserciones, el tiempo promedio por inserción es O(1) amortizado. 
>>>> No significa que cada inserción sea O(1), sino que el promedio de muchas inserciones consecutivas sí lo es.

**Ventajas**
+ Acceso aleatorio muy rápido.
+ Tamaño flexible.
+ Excelente aprovechamiento de la memoria caché.
+ Inserciones al final muy eficientes en promedio.
+ Implementación sencilla comparada con estructuras enlazadas.

**Desventajas**
+ Las inserciones en medio requieren desplazar elementos.
+ El crecimiento implica copiar todos los datos.
+ Puede reservar memoria adicional no utilizada (capacidad mayor que tamaño).

**Comparación con el Arreglo Estático**

| Característica     | Arreglo Estático | Arreglo Dinámico |
| ------------------ | ---------------- | ---------------- |
| Tamaño             | Fijo             | Variable         |
| Memoria            | Contigua         | Contigua         |
| Acceso por índice  | O(1)             | O(1)             |
| Inserción al final | O(1)*            | O(1) amortizado  |
| Redimensionamiento | No               | Sí               |

### Ideas clave 

+ Es una excelente elección cuando se necesita acceso frecuente por índice y el tamaño de la colección puede variar durante la ejecución del programa.
+ El Arreglo Dinámico combina la rapidez de acceso de un arreglo estático con la flexibilidad de crecer según sea necesario.
+ Su capacidad de redimensionar solo ocasionalmente hace que las inserciones al final sean muy eficientes, razón por la cual es la implementación predeterminada del TDA Lista en muchos lenguajes modernos.
