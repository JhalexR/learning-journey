# ¿Qué es una Lista Doblemente Enlazada?

Cada nodo mantiene dos enlaces:

+ Un enlace hacia el nodo siguiente
+ Un enlace hacia el nodo anterior

> es una evolución de la Lista Simplemente Enlazada.

**Estructura**
```
NULL ← [10] ⇄ [20] ⇄ [30] → NULL
        ↑                ↑
       head             tail

Cada nodo puede recorrer la lista en ambas direcciones:

10 → 20 → 30

y también:

30 → 20 → 10
```

**Estructura de un nodo**

Un nodo contiene
```
Nodo
├── anterior
├── dato
└── siguiente

ejemplo:

NULL ← [ prev | 10 | next ] ⇄ [ prev | 20 | next ] ⇄ [ prev | 30 | next ] → NULL
```
**Comparación con una Lista Simplemente Enlazada**

```
Lista simplemente enlazada 

head
 │
 ▼
[10] → [20] → [30] → NULL

Cada nodo solo conoce al siguiente 

Lista doblemente enlazada

head
 │
 ▼
NULL ← [10] ⇄ [20] ⇄ [30] → NULL
                                  ▲
                                 tail

Cada nodo conoce anterior - siguiente
```

### Referencias `head` y `tail`

```
head                          tail
 │                             │
 ▼                             ▼
[10] ⇄ [20] ⇄ [30] ⇄ [40]
```
Esto permite acceder directamente al:

+ primer nodo
+ último nodo

### Operaciones Básicas

**Insertar al inicio**

```
head
 │
 ▼
[20] ⇄ [30] ⇄ [40]

Queremos insertar 10.

Conceptualmente:

nuevo.next = head
head.prev = nuevo
head = nuevo

head
 │
 ▼
[10] ⇄ [20] ⇄ [30] ⇄ [40]

Complejidad: O(1)
```

**Insertar al final**

```
[10] ⇄ [20] ⇄ [30]
                     ▲
                    tail

Para insertar 40:

nuevo.prev = tail
tail.next = nuevo
tail = nuevo

[10] ⇄ [20] ⇄ [30] ⇄ [40]
                            ▲
                           tail

Complejidad: O(1)
```

**Insertar en medio**

```
[10] ⇄ [20] ⇄ [40]

Queremos insertar 30 entre 20 y 40.

Deben actualizarse cuatro enlaces:

20.next = 30
30.prev = 20

30.next = 40
40.prev = 30

[10] ⇄ [20] ⇄ [30] ⇄ [40]

Complejidad
> Si ya conocemos la posición donde insertar: O(1)
> Si necesitamos buscar primero esa posición: O(n)
```

**Eliminar un nodo**

```
[10] ⇄ [20] ⇄ [30] ⇄ [40]

Queremos eliminar 30

20.next = 40
40.prev = 20

[10] ⇄ [20] ⇄ [40]

Complejidad:
> Si ya conocemos el nodo que queremos eliminar: O(1)
> Si primero debemos buscarlo: O(n)
```

**Recorrido**

hacia adelante

+ Comenzando desde `head`

```
head
 │
 ▼
10 → 20 → 30 → 40 → NULL

Se utiliza el enlace: next
```

hacia atrás

+ Comenzando desde tail

```
tail
 │
 ▼
40 → 30 → 20 → 10 → NULL

Se utiliza el enlace: prev
```

> Esta capacidad es una de las grandes ventajas frente a la lista simplemente enlazada

**Búsqueda**

+ Para buscar un valor, todavía hay que recorrer la lista.
+ Aunque la lista sea doblemente enlazada, no existe acceso directo por índice.

Tiempo: O(n)

### Complejidad Temporal

| Operación                               | Complejidad |
| --------------------------------------- | :---------: |
| Acceder por índice                      |     O(n)    |
| Buscar                                  |     O(n)    |
| Insertar al inicio                      |     O(1)    |
| Insertar al final con `tail`            |     O(1)    |
| Eliminar el inicio                      |     O(1)    |
| Eliminar el final con `tail`            |     O(1)    |
| Insertar antes/después de nodo conocido |     O(1)    |
| Eliminar nodo conocido                  |     O(1)    |
| Recorrer hacia adelante                 |     O(n)    |
| Recorrer hacia atrás                    |     O(n)    |

**Aplicaciones**

Las listas doblemente enlazadas son útiles cuando necesitamos movernos en ambas direcciones.

+ Navegación hacia adelante y atrás
+ Reproductores multimedia
+ Historial de navegación

## Implementación de un deque

**Una Deque permite:**

+ Insertar/eliminar por el inicio
+ Insertar/eliminar por el final

## Una lista doblemente enlazada es una implementación natural para esta estructura.

### Ideas clave

+ La Lista Doblemente Enlazada es una estructura concreta basada en nodos donde cada nodo mantiene dos referencias, Esto permite recorrer la estructura en ambas direcciones.
+ Su principal ventaja frente a la lista simplemente enlazada es que permite eliminar e insertar nodos conocidos de forma más sencilla y eficiente
+ Su principal desventaja es que cada nodo necesita una referencia adicional, por lo que consume más memoria
