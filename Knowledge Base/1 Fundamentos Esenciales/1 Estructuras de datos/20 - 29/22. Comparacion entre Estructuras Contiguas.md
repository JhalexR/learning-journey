# Comparacion entre Estructuras Contiguas (Basadas en Arreglos)

Podemos entenderlas como una evolución conceptual:

```
Arreglo
   │
   ├── Arreglo Estático
   │       └── Tamaño fijo
   │
   ├── Arreglo Dinámico
   │       └── Tamaño variable
   │
   └── Arreglo Multidimensional
           └── Matriz
```

**La diferencia principal está en la cantidad de dimensiones y en si el tamaño puede cambiar.**

**Comparación general**

| Característica     | Arreglo Estático             | Arreglo Dinámico    | Matriz                               |
| ------------------ | ---------------------------- | ------------------- | ------------------------------------ |
| Dimensiones        | 1                            | 1                   | 2 o más                              |
| Tamaño             | Fijo                         | Variable            | Puede ser fijo o variable            |
| Organización       | Secuencial                   | Secuencial          | Filas y columnas                     |
| Acceso             | `A[i]`                       | `A[i]`              | `M[i][j]`                            |
| Acceso por índice  | O(1)                         | O(1)                | O(1)                                 |
| Memoria            | Contigua                     | Contigua            | Generalmente contigua                |
| Inserción al final | Limitada                     | O(1) amortizado     | Depende de la implementación         |
| Uso principal      | Colección de tamaño conocido | Colección que crece | Datos tabulares o multidimensionales |

### 1. Arreglo Estático

+ Tiene una dimensión y su tamaño se define al crearlo.
+ Su principal fortaleza: El acceso es: O(1)
+ Su principal limitación: No puede crecer

### 2. Arreglo Dinámico

+ También es un arreglo unidimensional, pero puede aumentar su capacidad automáticamente.
+ Ventaja principal: Puede crecer dinámicamente.

### 3. Matriz

Una matriz organiza los datos en dos o más dimensiones.

```
        Columnas
          0   1   2
       ┌───┬───┬───┐
Fila 0 │10 │20 │30 │
       ├───┼───┼───┤
Fila 1 │40 │50 │60 │
       ├───┼───┼───┤
Fila 2 │70 │80 │90 │
       └───┴───┴───┘
```

### La diferencia fundamental: la dimensión

```
Arreglo Estático

[ A ][ B ][ C ][ D ]
```
```
Arreglo Dinámico

[ A ][ B ][ C ][ D ][   ][   ]
              ↑
       puede crecer
```
```
Matriz

[ A ][ B ][ C ]
[ D ][ E ][ F ]
[ G ][ H ][ I ]
```
**La diferencia más importante es: Arreglo estático y arreglo dinámico son estructuras lineales de una dimensión; la matriz organiza los datos en múltiples dimensiones**

**Comparación de complejidad en acceso**
| Estructura       | Ejemplo   | Complejidad |
| ---------------- | --------- | ----------- |
| Arreglo estático | `A[i]`    | O(1)        |
| Arreglo dinámico | `A[i]`    | O(1)        |
| Matriz           | `M[i][j]` | O(1)        |

### ¿Cuándo usar cada una?

**Arreglo Estático**
<Br>
Úsalo cuando:
+ conoces el tamaño de antemano
+ el tamaño no cambiará
+ necesitas simplicidad y bajo costo adicional

**Arreglo Dinámico**
<Br>
Úsalo cuando:
+ el número de elementos puede cambiar
+ necesitas acceso rápido por índice
+ agregas frecuentemente elementos al final

**Matriz**
<Br>
Úsala cuando los datos tienen una estructura multidimensional:
+ Tablero de ajedrez → 8 × 8
+ Tablero de ajedrez → 8 × 8
+ Mapa → coordenada X × coordenada Y
+ Datos → tiempo × fila × columna

### Relación entre ellas
```
              Estructuras Contiguas
                       │
          ┌────────────┼────────────┐
          │            │            │
      Array        ArrayList      Matriz
      estático     / Vector
```
### Ideas clave

Las tres estructuras comparten la ventaja fundamental de los arreglos:

+ El acceso directo mediante índices es muy eficiente: O(1).

Pero cada una resuelve una necesidad diferente:

+ Array estático = fijo
+ Array dinámico = flexible
+ Matriz = multidimensional
