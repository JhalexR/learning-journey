# ¿Qué es un Nodo de Árbol Binario?

Un Nodo de Árbol Binario es una estructura de datos concreta que contiene:

1. Un dato.
2. Una referencia al hijo izquierdo.
3. Una referencia al hijo derecho.

### El concepto de hijo

Supongamos: 40

Creamos dos nodos: 20 y 70 

Luego conectamos:

```
        40
       /  \
     20    70
```
Decimos que:

+ 40 es el padre de 20
+ 40 es el padre de 70
+ 20 es hijo izquierdo
+ 70 es hijo derecho

No todos los nodos tienen dos hijos.

ejemplo:
```
        40
       /
     20

Nodo 40

dato = 40

izquierdo → Nodo 20

derecho → NULL

El puntero derecho vale: NULL porque no existe un hijo derecho.
```
### ¿Por qué se llama "binario"?
```
Nodo
├── dato
├── hijo_Izquierdo
└── hijo_Derecho
```

cada nodo puede tener como máximo dos hijos:

+ un hijo izquierdo
+ un hijo derecho.

Nunca puede tener tres hijos.

### Nodo Hoja

```
      40
     /  \
   20    70
```

los nodos 20 y 70 son hojas. 

```
Internamente:

izquierdo = NULL
derecho   = NULL
```

### Nodo Interno

```
       40
      /
    20
   /
 10
```

Los nodos: 40 , 20 -> son internos.
El nodo: 10 -> es hoja.

### La raíz

Todo árbol comienza con un nodo especial llamado:

+ raíz (root)

```
        50
       /  \
     20    80
```

La raíz es: 50

### ¿Cómo se conecta un árbol?

Cada nodo apunta a sus hijos.

```
        40
       /  \
     20    70
    / \
   10 30

Nodo 40
    izquierdo → Nodo20
    derecho → Nodo70

Nodo20
    izquierdo → Nodo10
    derecho → Nodo30

Nodo10
    izquierdo = NULL
    derecho = NULL
```

### Memoria

```
        50
       /  \
     20    80

Dirección 1000

dato = 50

izquierdo → 5000

derecho → 8000
```
> los nodos no necesitan estar contiguos en memoria

### ¿Por qué usar un árbol?

La diferencia más importante es la organización.

Un árbol representa una estructura jerárquica

```
        20
       /  \
     10    30
             \
              40
```

### Operaciones sobre un nodo

El nodo por sí solo no implementa operaciones complejas.

+ almacenar un dato
+ conectar un hijo izquierdo
+ conectar un hijo derecho

### El nodo es la unidad básica

Estos árboles utilizan el nodo de arbol binario 

+ Árbol Binario -> No tiene reglas especiales.
+ Árbol Binario de Búsqueda (ABB) -> izquierda < padre < derecha
+ AVL -> Debe mantenerse balanceado.
+ Rojo-Negro -> Cada nodo tiene un color.
+ Árbol de Expresión...
+ Heap Binario... 

> La diferencia entre ellos no está en la estructura física del nodo, sino en las reglas que siguen para organizarlo.

## Relación con los árboles

### Es importante distinguir entre el nodo y el árbol.

Nodo es solo una pieza.

Mientras que:
```
        50
       /  \
     20    80
    / \
   10 30
```
**es un árbol, construido conectando varios nodos.**

### Ideas clave

+ El Nodo de Árbol Binario es la unidad básica con la que se construyen todos los árboles binarios. Cada nodo almacena un dato y dos referencias
+ A diferencia de las listas enlazadas, donde cada nodo solo apunta al siguiente, un nodo de árbol puede ramificarse, creando una estructura jerárquica.
+ El Nodo de Árbol Binario no suele utilizarse de forma aislada; su propósito es servir como la pieza fundamental para construir todas las variantes de árboles binarios
