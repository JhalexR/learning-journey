# ¿Qué es una Lista Simplemente Enlazada?

Una Lista Simplemente Enlazada es una estructura de datos concreta formada por una secuencia de nodos, donde cada nodo contiene:

+ un dato
+ una referencia o puntero al siguiente nodo.

Podemos representarlo así:

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│ Dato: 10    │      │ Dato: 20    │      │ Dato: 30    │
│ Siguiente ──┼─────▶│ Siguiente ──┼─────▶│ Siguiente ──┼──▶ NULL
└─────────────┘      └─────────────┘      └─────────────┘
```

La lista mantiene una referencia al primer nodo, normalmente llamada: `Head`

```
head
 │
 ▼
[10 | next] → [20 | next] → [30 | next] → NULL
```
## El Nodo

El componente fundamental es el Nodo.

```
Nodo
├── dato
└── siguiente

┌──────────────┐
│ dato = 10    │
│ next = ──────┼─────▶
└──────────────┘
```

Un nodo no conoce necesariamente toda la lista, Solo conoce su propio dato y quién es el siguiente nodo.

### ¿Cómo se diferencia de un arreglo?

+ Arreglo -> utiliza posiciones contiguas de memoria
+ Una lista enlazada: puede estar dispersa
+ Aunque los nodos están físicamente separados, las referencias los conectan lógicamente:

```
Memoria

Dirección 1000 → [10 | 2500]

Dirección 2500 → [20 | 8000]

Dirección 8000 → [30 | NULL]

[10] ─────▶ [20] ─────▶ [30] ─────▶ NULL
```

**Por eso se dice que está enlazada.**

### La referencia `head`

+ La lista necesita saber dónde comienza
+ Si perdemos head, perdemos el acceso al primer nodo y, por consecuencia, a toda la lista

```
head
 │
 ▼
[10] → [20] → [30] → NULL
```

### La referencia NULL

+ No existe un nodo siguiente.
+ Esto permite saber cuándo termina la lista.

### Operaciones fundamentales

Una Lista Simplemente Enlazada puede implementar las operaciones del TDA Lista.

**Insertar al inicio**

```
head
 │
 ▼
[20] → [30] → NULL

nuevo.next = head
head = nuevo

head
 │
 ▼
[10] → [20] → [30] → NULL

Complejidad O(1)
```

**Insertar al final**

```
[10] → [20] → [30] → NULL
[30].next = nuevo
[10] → [20] → [30] → [40] → NULL

Complejidad
Si solo tenemos head -> O(n)
Si mantenemos también una referencia tail -> Si mantenemos también una referencia tail
```

**Insertar después de un nodo**

```
[10] → [20] → [40]

Queremos insertar 30 después de 20.

nuevo.next = 20.next
20.next = nuevo

[10] → [20] → [30] → [40]

> La modificación de enlaces es O(1).
> Pero encontrar el nodo 20 puede costar O(n).
```

**Eliminar el primer nodo**

```
head
 │
 ▼
[10] → [20] → [30]

Para eliminar 10:

head = head.next

head
 │
 ▼
[20] → [30] → NULL

Complejidad -> O(1)
```
**Eliminar un nodo intermedio**

```
[10] → [20] → [30] → [40]

Queremos eliminar 30.

Necesitamos que 20 apunte directamente a 40.

[10] → [20] → [40] → NULL

Complejidad

> primero debemos localizar el nodo anterior, lo que normalmente requiere: O(n)
> La modificación del enlace es: O(1)
```
**Buscar un elemento**

Para buscar 30:

```
head
 │
 ▼
[10] → [20] → [30] → [40]
 ▲
 │
 Busqueda

 head
 │
 ▼
[10] → [20] → [30] → [40]
        ▲
        │
    Busqueda

 head
 │
 ▼
[10] → [20] → [30] → [40]
                ▲
                │
            Busqueda

Complejidad: O(n) -> Se recorre secuencialmente
```

**Acceso por índice**

+ no existe un acceso directo real
+ Hay que recorrer todo

```
lista[3]

head
 ↓
Nodo 0
 ↓
Nodo 1
 ↓
Nodo 2
 ↓
Nodo 3

Complejidad: O(n)
``` 

### Complejidad general

| Operación                            | Lista Simplemente Enlazada |
| ------------------------------------ | :------------------------: |
| Acceder por índice                   |            O(n)            |
| Buscar                               |            O(n)            |
| Insertar al inicio                   |            O(1)            |
| Eliminar al inicio                   |            O(1)            |
| Insertar después de un nodo conocido |            O(1)            |
| Eliminar después de un nodo conocido |            O(1)            |
| Insertar al final sin `tail`         |            O(n)            |
| Insertar al final con `tail`         |            O(1)            |

### Ventaja fundamental

**La gran ventaja de una lista enlazada es que no necesita mover todos los elementos cuando se insertan o eliminan nodos.**

```
[10] → [20] → [30]

Insertar 5:

[5] ───▶ [10] → [20] → [30]
```
Solo se modifica un enlace.

### Desventaja fundamental

La lista no permite saltar directamente a una posición.

```
Lista:

head
 ↓
Nodo 0 → Nodo 1 → Nodo 2 → ... → Nodo 999

A[999]

Nodo 0 → Nodo 1 → Nodo 2 → ... → Nodo 999

hay que recorrer todos los nodos anteriores.

```

### Implementación conceptual

```
Lista
  │
  └── head
        │
        ▼
      Nodo
      ├── dato: 10
      └── next ───▶ Nodo
                    ├── dato: 20
                    └── next ───▶ NULL
```

### Ideas clave

+ La Lista Simplemente Enlazada está formada por nodos conectados mediante referencias:
+ Su principal ventaja es que las inserciones y eliminaciones pueden ser muy eficientes porque solo requieren modificar enlaces: O(1)
+ Su principal desventaja es que no existe acceso aleatorio. Para llegar a una posición determinada hay que recorrer la lista desde el principio: O(n)
