# ¿Qué es una Lista Circular?

Su característica principal es que el último nodo no apunta a NULL, sino que vuelve a apuntar al primero.

> Una lista circular no tiene un final natural; después del último elemento se vuelve al primero.

**Estructura básica**

```
[10] → [20] → [30]
 ↑                 │
 └─────────────────┘
```
```
head
 │
 ▼
[10] → [20] → [30]
 ↑                 │
 └─────────────────┘
```
**Lista Circular con tail**

Una implementación muy común mantiene una referencia al último nodo:

```
tail
 │
 ▼
[30]
 │
 └── next → [10] → [20] → [30]

 tail.next
    │
    ▼
  [10] → [20] → [30]
             ▲
             │
            tail
```

Esto permite obtener rápidamente tanto:

+ el primer nodo
+ el último nodo

**Insertar al inicio**

```
[10] → [20] → [30]
 ↑                 │
 └─────────────────┘

 Insertamos 5.

 [5] → [10] → [20] → [30]
 ↑                       │
 └───────────────────────┘

 Si mantenemos tail, basta con actualizar:

nuevo.next = tail.next
tail.next = nuevo

Como tail.next era el primer nodo, ahora el nuevo nodo se convierte en el primero.

Complejidad -> O(1)
```

**Insertar al final**

```
[10] → [20] → [30]
 ↑                 │
 └─────────────────┘

 Insertamos 40.

 [10] → [20] → [30] → [40]
 ↑                       │
 └───────────────────────┘

nuevo.next = tail.next
tail.next = nuevo
tail = nuevo

Complejidad con tail -> O(1)
```

**Eliminar el primer nodo**

```
[10] → [20] → [30]
 ↑                 │
 └─────────────────┘

 Eliminamos 10.

 [20] → [30]
 ↑          │
 └──────────┘

 Si tenemos tail:

 tail.next = tail.next.next

 Complejidad -> O(1)
```
**Eliminar el último nodo**

```
[10] → [20] → [30]
 ↑                 │
 └─────────────────┘

 Para eliminar 30, necesitamos que 20 apunte a 10:

 [10] → [20]
 ↑          │
 └──────────┘

 > Pero para encontrar el nodo anterior al último debemos recorrer la lista.

 Complejidad -> O(n)
```

## Lista Circular Doblemente Enlazada

También podemos combinar circularidad con doble enlace.

```
       ┌─────────────────────┐
       │                     │
       ▼                     │
[10] ⇄ [20] ⇄ [30] ⇄ [40]
 ↑                         │
 └─────────────────────────┘

head.prev = tail
tail.next = head

Esto permite recorrer:

Hacia adelante

10 → 20 → 30 → 40 → 10 → 20 → ...

Hacia atrás

40 → 30 → 20 → 10 → 40 → 30 → ...
```
> Es una estructura especialmente flexible

### Complejidad Temporal 

> Para una lista circular simplemente enlazada con tail

| Operación              | Complejidad |
| ---------------------- | :---------: |
| Acceder por índice     |     O(n)    |
| Buscar                 |     O(n)    |
| Insertar al inicio     |     O(1)    |
| Insertar al final      |     O(1)    |
| Eliminar inicio        |     O(1)    |
| Eliminar final         |     O(n)    |
| Recorrer toda la lista |     O(n)    |

**Aplicaciones**


+ Planificación Round Robin
+ Turnos
+ Reproductores musica
+ Buffers circulares
    + sistemas de entrada/salida
    + streaming
    + procesamiento de datos
    + comunicación entre procesos

> Round Robin: es un método en el que un grupo de elementos (personas, tareas, equipos o procesos) se turnan de manera equitativa y cíclica. Se utiliza comúnmente para repartir trabajo equitativamente, programar partidos deportivos, organizar ideas en grupo

### Ideas clave

+ Una lista normal representa una secuencia con principio y final; una lista circular representa una secuencia que puede repetirse indefinidamente.
+ es especialmente útil para turnos, planificación cíclica, buffers circulares, reproducción repetida y cualquier problema donde después del último elemento se deba volver al primero.
