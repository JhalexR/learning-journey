# Comparacion entre Estructuras Enlazadas (Basadas en Punteros)

Se diferencian principalmente por la cantidad de enlaces por nodo y si el último elemento se conecta con el primero.

| Característica        | Simplemente Enlazada | Doblemente Enlazada      | Circular              |
| --------------------- | -------------------- | ------------------------ | --------------------- |
| `next`                | Sí                   | Sí                       | Sí                    |
| `prev`                | No                   | Sí                       | Opcional              |
| Último → primero      | No                   | No necesariamente        | Sí                    |
| Recorrido hacia atrás | No                   | Sí                       | Solo con `prev`       |
| Memoria por nodo      | Baja                 | Mayor                    | Similar a su variante |
| Mejor para            | Secuencias simples   | Navegación bidireccional | Procesos cíclicos     |

### 1. Estructura Enlazada Simple (Singly Linked List)

Cada nodo contiene los datos y un único puntero que apunta al nodo siguiente. El último nodo apunta a NULL

+ Ventajas: Es la más simple de implementar y requiere menor memoria por nodo al almacenar solo un puntero.

+ Desventajas: Solo se puede recorrer en una dirección (hacia adelante). Para llegar a un nodo anterior, es necesario reiniciar el recorrido desde el nodo cabeza.

+ Uso ideal: Pilas (Stacks) y listas dinámicas básicas donde los elementos solo se procesan secuencialmente en un sentido

### 2. Estructura Doblemente Enlazada (Doubly Linked List)

Cada nodo contiene los datos, un puntero al nodo siguiente y un puntero al nodo anterior

+ Ventajas: Permite la navegación bidireccional (hacia adelante y hacia atrás). Eliminar o insertar un nodo es más eficiente porque no necesitas un registro previo del predecesor.
+ Desventajas: Consume más memoria al duplicar los punteros por cada nodo. La gestión de modificar dos punteros en lugar de uno aumenta la complejidad del código.
+ Uso ideal: Aplicaciones que requieren deshacer/rehacer acciones (Ctrl+Z) y navegadores web.

### 3. Estructura Circular (Circular Linked List)

**Puede ser simple o doble**. La característica clave es que no hay un puntero nulo final. El puntero del último nodo enlaza directamente con el primer nodo de la lista

+ Ventajas: Puedes recorrer toda la estructura comenzando desde cualquier nodo. Si es doblemente enlazada, el primer nodo enlaza hacia atrás con el último nodo, haciendo que recorrer los extremos sea instantáneo.
+ Desventajas: Si no se controla bien el ciclo de recorrido, existe el riesgo de caer en bucles infinitos. Requiere más cuidado en las inserciones y eliminaciones para no romper la continuidad del círculo.
+ Uso ideal: Turnos rotativos en videojuegos multijugador, búferes circulares y listas de reproducción continuas de música
