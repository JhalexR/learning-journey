# ¿Qué es una Tabla Hash?

Una Tabla Hash almacena elementos utilizando una función hash que transforma una clave en una posición dentro de un arreglo.

La idea general es:

```
Clave
  │
  ▼
Función Hash
  │
  ▼
Índice del arreglo
  │
  ▼
Valor almacenado
```
**En lugar de recorrer todos los elementos buscando a Juan, calculamos directamente dónde debería encontrarse.**

Ejemplo:

```
Clave: "Juan"
       │
       ▼
   hash("Juan")
       │
       ▼
      4
       │
       ▼
tabla[4] = "Información de Juan"
```

### Estructura básica

Una Tabla Hash puede visualizarse como un arreglo:

```
Índice

  0   ┌─────────────┐
      │             │
  1   ┌─────────────┐
      │             │
  2   ┌─────────────┐
      │             │
  3   ┌─────────────┐
      │             │
  4   ┌─────────────┐
      │ "Juan"      │
  5   ┌─────────────┐
      │             │
  6   ┌─────────────┐
      │             │
```

**La función hash determina:**
+ "Juan" → 4 
+ tabla[4] -> es la posición donde se almacena o busca esa clave.

## La Función `Hash`

La función hash recibe una clave y produce un número entero.

Conceptualmente:

```
hash(clave) → número

Pero ese número debe convertirse en un índice válido:

    índice = hash(clave) % capacidad
```
**Ejemplo sencillo**
```
Supongamos una tabla con capacidad 10.

Queremos almacenar: 
    "Juan" → 25 años

    La función hash produce:
        hash("Juan") = 37

        Calculamos:
        37 % 10 = 7

            Entonces:
            tabla[7] = ("Juan", 25)
```

Visualmente:
```
Índice

0  ┌─────────┐
1  ┌─────────┐
2  ┌─────────┐
3  ┌─────────┐
4  ┌─────────┐
5  ┌─────────┐
6  ┌─────────┐
7  │Juan, 25 │
8  ┌─────────┐
9  ┌─────────┐
```

## Operaciones fundamentales

Una Tabla Hash normalmente ofrece operaciones como:

+ put(clave, valor)
+ get(clave)
+ remove(clave)
+ containsKey(clave)

Por ejemplo:
+ put("Juan", 25)
+ get("Juan")
+ Devuelve -> 25

**Insertar**

```
Supongamos:
put("Ana", 30)
hash("Ana") → índice 3
tabla[3] = ("Ana", 30)
```

**Buscar**

Cuando hacemos:

+ get("Ana")

la tabla:

1. calcula el hash de "Ana"
2. obtiene el mismo índice
3. accede directamente a esa posición
4. devuelve el valor

```
"Ana"
  │
  ▼
hash("Ana")
  │
  ▼
  3
  │
  ▼
tabla[3]
```

En condiciones ideales: O(1)

### Factor de carga

Un concepto importante es el factor de carga:

+ factor de carga = elementos almacenados / capacidad de la tabla

Por ejemplo:

```
Elementos = 7, Capacidad = 10
factor de carga = 7 / 10
factor de carga = 0.7
```
Cuando la tabla se llena demasiado, aumentan las _**colisiones.**_

_**Una colisión ocurre cuando dos claves diferentes producen el mismo índice**_

## El problema de las colisiones

Esto es inevitable en muchas tablas hash porque normalmente tenemos:

+ Muchas claves posibles
+ Pero Un número limitado de posiciones

Por ejemplo:
```
hash("Ana")  → 3
hash("Luis") → 3

Ambas claves quieren ocupar:

tabla[3]
```
# Resolución de colisiones

> Necesitamos una estrategia para resolver las colisiones.
```
Resolución de colisiones
│
├── Encadenamiento (Separate Chaining)
│
└── Direccionamiento Abierto (Open Addressing)
│    │
│    ├── Sondeo Lineal (Linear Probing)
│    ├── Sondeo Cuadrático (Quadratic Probing)
│    └── Doble Hashing (Double Hashing)
│
└── Redimensionamiento
```
1. Primero se aplica una estrategia de resolución de colisiones para encontrar un lugar libre. 

2. Mientras que las estrategias de resolucion de colisiones ocurren de manera inmediata, en el milisegundo en que el espacio original está ocupado.

3. El dimensionamiento ocurre de forma reactiva, cuando se alcanza un factor de carga predeterminado, normalmente _**∂ = 0.75**_

### Resolución de colisiones: Encadenamiento (_**Hash Chaining ó Separate Chaining**_)

Una de las estrategias más comunes es el encadenamiento separado (Separate Chaining).

```
0 → NULL
1 → ("Pedro", 20)
2 → NULL
3 → ("Ana", 30)
4 → ("María", 28)
5 → NULL

hash("Luis, 25")  → 3 → Colision 

0 → NULL
1 → ("Pedro", 20)
2 → NULL
3 → ("Ana", 30) → ("Luis", 25)
4 → ("María", 28)
5 → NULL
```
_**La tabla utiliza una estructura secundaria, normalmente una lista enlazada, para almacenar las colisiones.**_

## Resolución de colisiones: Direccionamiento Abierto

+ El direccionamiento abierto mantiene todos los elementos dentro del propio arreglo.
+ Si una posición está ocupada, se busca otra posición disponible.

Por ejemplo:

```
hash("Ana") = 3

Pero:
    tabla[3] está ocupada
    
    Entonces se busca:
                    tabla[4]

                    Si también está ocupada:
                                            tabla[5]
```
Esto se llama direccionamiento abierto (Open Addressing).

### Direccionamiento Abierto: Sondeo lineal

+ La estrategia más sencilla es el sondeo lineal:

+ índice original = 3
+ Si otra clave produce el índice 3, se busca la siguiente posición libre.
+ 3 → 4 → 5 → 6 → ...

> Si la posición está ocupada, revisar la siguiente, luego la siguiente, y así sucesivamente.

### Direccionamiento Abierto: Sondeo Cuadrático

+ Aquí los saltos aumentan cuadráticamente.

Por ejemplo:

```
3

↓

4 (+1²)

↓

7 (+2²)

↓

12 (+3²)
```

+ En una tabla de tamaño fijo, siempre aplicando módulo a la capacidad.
+ La idea es evitar que muchos elementos se agrupen consecutivamente.

### Direccionamiento Abierto: Doble Hashing

En lugar de usar un incremento fijo, se calcula un segundo hash.
```
nuevo índice = hash1 + i × hash2
```
Esto produce secuencias diferentes para distintas claves y reduce aún más las agrupaciones.

### Resumen

```
Resolución de colisiones
│
├── Encadenamiento
│     └── Cada posición tiene una lista
│
└── Direccionamiento Abierto
      │
      ├── Sondeo Lineal
      │      3 → 4 → 5 → 6
      │
      ├── Sondeo Cuadrático
      │      3 → 4 → 7 → ...
      │
      └── Doble Hashing
             3 → 8 → 2 → ...
```

### Comparación

| Característica                                    | Direccionamiento Abierto | Sondeo Lineal                |
| ------------------------------------------------- | ------------------------ | ---------------------------- |
| ¿Qué es?                                          | Estrategia general       | Técnica específica           |
| ¿Resuelve colisiones?                             | Sí                       | Sí                           |
| ¿Mantiene todos los elementos dentro del arreglo? | Sí                       | Sí                           |
| ¿Cómo busca otra posición?                        | Depende del método       | Avanza una posición cada vez |
| ¿Es el único método?                              | No                       | Sí, es uno de varios         |


## Redimensionamiento

Supongamos: 

```
Capacidad = 10
Elementos = 8

La tabla puede aumentar su capacidad:

10 → 20

Pero no basta con copiar los elementos directamente.

¿Por qué?

Porque el índice depende de:

hash(clave) % capacidad

Antes:
    hash("Ana") % 10

Después:
    hash("Ana") % 20

```
El resultado puede ser diferente.

Por eso hay que volver a calcular la posición de cada elemento.

Este proceso se llama: _**Rehashing**_

### Complejidad

| Operación | Promedio | Peor caso |
| --------- | :------: | :-------: |
| Insertar  |   O(1)   |    O(n)   |
| Buscar    |   O(1)   |    O(n)   |
| Eliminar  |   O(1)   |    O(n)   |

> En condiciones ideales

El peor caso puede ocurrir si muchas claves terminan en la misma posición:

```
tabla[3]
   │
   ▼
A → B → C → D → E → F
```
### Ventajas

La Tabla Hash ofrece:

+ búsquedas muy rápidas en promedio
+ inserciones rápidas
+ eliminaciones rápidas
+ acceso mediante claves arbitrarias
+ flexibilidad para almacenar pares clave-valor.

### Desventajas

No es perfecta.

Sus principales desventajas son:

+ puede haber colisiones
+ depende de una buena función hash
+ puede requerir redimensionamiento
+ consume memoria adicional
+ no mantiene necesariamente los elementos ordenados.

### Ideas clave

+ La Tabla Hash es una estructura concreta que utiliza una función hash para transformar una clave en una posición de un arreglo.
+ Su objetivo principal es lograr: Buscar, Insertar, Eliminar en tiempo promedio O(1)
+ La Tabla Hash es una de las implementaciones concretas más importantes del TDA Diccionario/Mapa.
