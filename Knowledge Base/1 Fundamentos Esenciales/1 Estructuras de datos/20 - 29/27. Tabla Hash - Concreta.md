# ¿Qué es una Tabla Hash?

Una Tabla Hash almacena elementos utilizando una función hash que transforma una clave en una posición dentro de un arreglo.

La idea general es:

```
Clave
  │
  ▼
Función Hash
  │
  ▼
Índice del arreglo
  │
  ▼
Valor almacenado
```
**En lugar de recorrer todos los elementos buscando a Juan, calculamos directamente dónde debería encontrarse.**

Ejemplo:

```
Clave: "Juan"
       │
       ▼
   hash("Juan")
       │
       ▼
      4
       │
       ▼
tabla[4] = "Información de Juan"
```

### Estructura básica

Una Tabla Hash puede visualizarse como un arreglo:

```
Índice

  0   ┌─────────────┐
      │             │
  1   ┌─────────────┐
      │             │
  2   ┌─────────────┐
      │             │
  3   ┌─────────────┐
      │             │
  4   ┌─────────────┐
      │ "Juan"      │
  5   ┌─────────────┐
      │             │
  6   ┌─────────────┐
      │             │
```

**La función hash determina:**
+ "Juan" → 4 
+ tabla[4] -> es la posición donde se almacena o busca esa clave.

## La Función `Hash`

La función hash recibe una clave y produce un número entero.

Conceptualmente:

```
hash(clave) → número

Pero ese número debe convertirse en un índice válido:

    índice = hash(clave) % capacidad
```
**Ejemplo sencillo**
```
Supongamos una tabla con capacidad 10.

Queremos almacenar: 
    "Juan" → 25 años

    La función hash produce:
        hash("Juan") = 37

        Calculamos:
        37 % 10 = 7

            Entonces:
            tabla[7] = ("Juan", 25)
```

Visualmente:
```
Índice

0  ┌─────────┐
1  ┌─────────┐
2  ┌─────────┐
3  ┌─────────┐
4  ┌─────────┐
5  ┌─────────┐
6  ┌─────────┐
7  │Juan, 25 │
8  ┌─────────┐
9  ┌─────────┐
```

## Operaciones fundamentales

Una Tabla Hash normalmente ofrece operaciones como:

+ put(clave, valor)
+ get(clave)
+ remove(clave)
+ containsKey(clave)

Por ejemplo:
+ put("Juan", 25)
+ get("Juan")
+ Devuelve -> 25

**Insertar**

```
Supongamos:
put("Ana", 30)
hash("Ana") → índice 3
tabla[3] = ("Ana", 30)
```

**Buscar**

Cuando hacemos:

+ get("Ana")

la tabla:

1. calcula el hash de "Ana"
2. obtiene el mismo índice
3. accede directamente a esa posición
4. devuelve el valor

```
"Ana"
  │
  ▼
hash("Ana")
  │
  ▼
  3
  │
  ▼
tabla[3]
```

En condiciones ideales: O(1)

## El problema de las colisiones

Una colisión ocurre cuando dos claves diferentes producen el mismo índice

Esto es inevitable en muchas tablas hash porque normalmente tenemos:
+ Muchas claves posibles
+ Pero Un número limitado de posiciones

Por ejemplo:
```
hash("Ana")  → 3
hash("Luis") → 3

Ambas claves quieren ocupar:

tabla[3]
```
