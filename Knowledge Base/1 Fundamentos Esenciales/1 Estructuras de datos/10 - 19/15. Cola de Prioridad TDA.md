# ¿Qué es una Cola de Prioridad (Priority Queue)?

La Cola de Prioridad (Priority Queue) pertenece a las estructuras abstractas de búsqueda porque los elementos no se procesan por orden de llegada, sino según una prioridad asociada.

Una Cola de Prioridad es un Tipo de Dato Abstracto (TDA) que almacena elementos junto con una prioridad y siempre elimina primero el elemento con la mayor o menor prioridad (según cómo se defina).

A diferencia de una cola FIFO, el orden de salida depende de la prioridad, no del momento de inserción.

#### Analogía

Imagina la sala de urgencias de un hospital.

Llegan los pacientes en este orden:
```
Juan   (Prioridad 3)
Ana    (Prioridad 1)
Pedro  (Prioridad 2)
```
Aunque Juan llegó primero, el orden de atención será:
```
Ana   (Prioridad 1)
Pedro (Prioridad 2)
Juan  (Prioridad 3)
```

_**La prioridad determina quién sale primero.**_

Ejemplo:

Supongamos una cola donde un número menor significa mayor prioridad:

```
Elemento     Prioridad
A                 5
B                 2
C                 8
D                 1

[(A,5), (B,2), (C,8), (D,1)]

El siguiente elemento en salir será:
> D

porque tiene la prioridad más alta (1).
```
#### Operaciones básicas

| Operación   | Descripción                                         |
| ----------- | --------------------------------------------------- |
| Crear       | Construir una cola vacía.                           |
| Insert(x,p) | Insertar un elemento con prioridad `p`.             |
| Peek()      | Consultar el elemento de mayor prioridad.           |
| Extract()   | Eliminar y devolver el elemento de mayor prioridad. |
| isEmpty()   | Verificar si está vacía.                            |
| Size()      | Número de elementos.                                |

Algunas implementaciones también permiten:
+ ChangePriority()
+ Remove()
+ Merge()

#### NO es una cola normal

Comparemos:

+ **Cola (FIFO):**

```
Llegan

A
B
C
D

↓

Salen

A
B
C
D
```

+ **Cola de Prioridad**

```
Llegan

A (5)
B (2)
C (8)
D (1)

↓

Salen

D
B
A
C
```
**La prioridad rompe el orden de llegada.**

**Ejemplo**

Comenzamos con una cola vacía.

Insertamos:
```
Insert(A,5)
Insert(B,2)
Insert(C,8)
Insert(D,1)
```
Conceptualmente queda:

```
[(A,5), (B,2), (C,8), (D,1)]
```
> ### Aunque ese sea el orden de inserción, internamente la estructura organiza los elementos para que el siguiente en salir sea el de mayor prioridad.

```
Peek()

↓

D
```
```
Extract()

↓

D
```
Ahora el siguiente será:

```
B
```
### Tipos de cola de prioridad

#### Min Priority Queue

Siempre extrae el elemento con el valor de prioridad más pequeño.

```
Prioridades

7
2
5
1

↓

Sale primero

1
```

#### Max Priority Queue

Siempre extrae el elemento con el valor de prioridad más grande.
```
Prioridades

7
2
5
1

↓

Sale primero

7
```

### Implementaciones concretas

#### 1. Heap (Montículo)

Es la implementación más utilizada.

**Min Heap**

```
        1
      /   \
     2     5
    / \
   8  10
```
La raíz siempre contiene el elemento de mayor prioridad (el menor valor en un min-heap).

#### Complejidad temporal:
| Operación | Tiempo   |
| --------- | -------- |
| Insert    | O(log n) |
| Peek      | O(1)     |
| Extract   | O(log n) |

#### 2. Lista ordenada

**Los elementos se mantienen ordenados.**

```
1 → 2 → 5 → 8
```

+ Extraer el primero es muy rápido.
+ Insertar requiere encontrar la posición correcta.

#### Complejidad temporal:
| Operación | Tiempo |
| --------- | ------ |
| Insert    | O(n)   |
| Peek      | O(1)   |
| Extract   | O(1)   |

#### 3. Lista desordenada

```
5 → 2 → 8 → 1
```

+ Insertar es inmediato.
+ Para extraer el mejor elemento hay que recorrer toda la lista.

#### Complejidad temporal:
| Operación | Tiempo |
| --------- | ------ |
| Insert    | O(1)   |
| Peek      | O(n)   |
| Extract   | O(n)   |

#### Comparación entre implementaciones

| Implementación    |  Insert  | Peek |  Extract |
| ----------------- | :------: | :--: | :------: |
| Lista desordenada |   O(1)   | O(n) |   O(n)   |
| Lista ordenada    |   O(n)   | O(1) |   O(1)   |
| Heap              | O(log n) | O(1) | O(log n) |

**El heap ofrece el mejor equilibrio para la mayoría de aplicaciones.**

#### Aplicaciones

Las colas de prioridad se usan cuando siempre debe atenderse el elemento "más importante":

+ planificadores de procesos de un sistema operativo
+ algoritmos de caminos mínimos como Dijkstra
+ búsqueda informada como A*
+ simulaciones de eventos discretos
+ programación de tareas
+ sistemas de impresión con prioridades

#### Ideas claves

**La cola de prioridad conserva la idea de una cola (insertar y extraer elementos), pero reemplaza la regla FIFO por una regla de prioridad.**

**En la práctica, casi siempre se implementa mediante un heap, ya que permite mantener eficientemente el elemento prioritario siempre disponible para su consulta o extracción**
