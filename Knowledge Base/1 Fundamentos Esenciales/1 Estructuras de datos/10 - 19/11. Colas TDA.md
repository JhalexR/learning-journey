# ¿Qué es una cola (Queue)?

Una cola es un Tipo de Dato Abstracto (TDA) que almacena una colección ordenada de elementos donde:

+ Los nuevos elementos se insertan al final (cola o rear).
+ Los elementos se eliminan del frente (front).

Su funcionamiento sigue el principio FIFO (First In, First Out) -> El primer elemento que entra es el primero que sale.

#### Analogía

una fila para comprar boletos

```
Frente                           Final
  │                                │
  ▼                                ▼
[Ana] → [Luis] → [María] → [Pedro]
```

+ Pedro es el último en llegar.
+ Ana es la primera en ser atendida.

> _**Nadie puede "colarse" en la mitad de la fila.**_
>> _**Ese es exactamente el comportamiento de una cola.**_

#### Operaciones básicas

Un TDA Cola suele definir las siguientes operaciones

| Operación        | Descripción                                  |
| ---------------- | -------------------------------------------- |
| Crear            | Construir una cola vacía.                    |
| Enqueue(x)       | Insertar un elemento al final.               |
| Dequeue()        | Eliminar y devolver el elemento del frente.  |
| Front() o Peek() | Consultar el primer elemento sin eliminarlo. |
| isEmpty()        | Verificar si la cola está vacía.             |
| Size()           | Obtener el número de elementos.              |

#### Enqueue (Agregar un elemento al final)
```
Front          Rear
  │             │
  ▼             ▼
10 → 20 → 30

> Enqueue(40)

Front               Rear
  │                  │
  ▼                  ▼
10 → 20 → 30 → 40

```

#### Dequeue (Eliminar el primer elemento)
```
Front               Rear
  │                  │
  ▼                  ▼
10 → 20 → 30 → 40

> Dequeue()

Front          Rear
  │             │
  ▼             ▼
20 → 30 → 40
```

#### Front o Peek (Consultar el primer elemento sin eliminarlo)
```
Front          Rear
  │             │
  ▼             ▼
20 → 30 → 40

> Front()
> 20
```
## Implementaciones

Puede implementarse mediante:

+ arreglos
+ listas enlazadas
+ arreglos circulares

#### 1. Con arreglo

Para evitar mover elementos tras cada Dequeue, normalmente se mantienen dos índices:

+ **front**
+ **rear**

```
Índice

0   1   2   3
┌───┬───┬───┬───┐
│10 │20 │30 │40 │
└───┴───┴───┴───┘
 ↑             ↑
Front        Rear
```
_**En implementaciones eficientes se utiliza un arreglo circular, reutilizando posiciones libres.**_

#### 2. Con lista enlazada

La cola mantiene dos referencias:

+ una al primer nodo (front)
+ otra al último (rear)

```
Front                      Rear
  │                          │
  ▼                          ▼
┌─────────┐    ┌─────────┐    ┌──────────┐
│10 | •───┼──► │20 | •───┼──► │30 | NULL │
└─────────┘    └─────────┘    └──────────┘
```

+ Enqueue agrega un nodo al final.
+ Dequeue elimina el primer nodo.

| Operación | Arreglo circular | Lista enlazada |
| --------- | :--------------: | :------------: |
| Enqueue   |       O(1)       |      O(1)      |
| Dequeue   |       O(1)       |      O(1)      |
| Front     |       O(1)       |      O(1)      |
| isEmpty   |       O(1)       |      O(1)      |


#### Aplicaciones comunes

Las colas se utilizan cuando los elementos deben procesarse en el mismo orden en que llegan:

+ Atención de clientes.
+ Impresión de documentos.
+ Planificación de procesos en sistemas operativos.
+ Buffers de entrada y salida.
+ Recorridos por niveles de árboles (Breadth-First Search, BFS).
+ Simulación de líneas de espera.

### Bicola (Deque)

La bicola (Deque, Double-Ended Queue) puede considerarse una generalización de la cola. Comparte la misma idea básica (una secuencia ordenada de elementos), pero elimina una de las restricciones de la cola tradicional.

**Otra forma de verlo es que la cola es un caso particular de la bicola.**

#### ¿Cuál es la diferencia?

#### Cola (Queue)

Solo permite:

+ insertar al final.
+ eliminar del frente.

```
Enqueue  ─────────►

Front                 Rear
  │                    │
  ▼                    ▼
10 → 20 → 30 → 40

◄──────── Dequeue
```
Sigue la política FIFO.

#### Bicola (Deque)

Permite operar en ambos extremos.

```
Insertar ◄────────────► Insertar

Front                 Rear
  │                    │
  ▼                    ▼
10 ⇄ 20 ⇄ 30 ⇄ 40

Eliminar ◄────────────► Eliminar
```
Es decir, permite:

+ insertar al frente
+ insertar al final
+ eliminar del frente
+ eliminar del final

La única diferencia es que una cola restringe las operaciones a un solo sentido, mientras que una bicola permite operar en ambos extremos.

#### Operaciones típicas

| Operación      | Descripción                  |
| -------------- | ---------------------------- |
| InsertFront(x) | Inserta al frente.           |
| InsertRear(x)  | Inserta al final.            |
| RemoveFront()  | Elimina el primer elemento.  |
| RemoveRear()   | Elimina el último elemento.  |
| Front()        | Consulta el primer elemento. |
| Rear()         | Consulta el último elemento. |
| isEmpty()      | Verifica si está vacía.      |
| Size()         | Número de elementos.         |

### Implementación de Bicola

La implementación más natural es una lista doblemente enlazada.

```
NULL ← [10] ⇄ [20] ⇄ [30] ⇄ [40] → NULL
```
Como cada nodo conoce a su anterior y a su siguiente:

+ insertar al frente es O(1),
+ insertar al final es O(1),
+ eliminar al frente es O(1),
+ eliminar al final es O(1).

> También puede implementarse con un arreglo circular, aunque requiere un manejo más cuidadoso de los índices.

#### Ideas claves

_**La cola restringe las operaciones para preservar el orden de llegada: el primer elemento que entra es el primero que sale (FIFO). Esta propiedad la hace ideal para modelar procesos donde los elementos deben atenderse de forma secuencial y justa, como filas de personas, tareas pendientes o solicitudes en un sistema.**_

_**Suele decirse que: Una cola es una bicola con restricciones.**_
