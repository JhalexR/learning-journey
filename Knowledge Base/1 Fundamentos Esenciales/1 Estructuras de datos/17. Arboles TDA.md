# ¿Qué es un Árbol (Tree)?

Un Árbol es un Tipo de Dato Abstracto (TDA) que organiza elementos llamados nodos mediante relaciones de padre e hijos, formando una estructura jerárquica.

Su característica principal es:

+ Existe un único camino desde la raíz hasta cualquier nodo.

**Analogía**

el organigrama de una empresa.

```
Director
├── Gerente A
│   ├── Empleado 1
│   └── Empleado 2
└── Gerente B
    ├── Empleado 3
    └── Empleado 4
```
Existe una relación jerárquica entre los elementos.

**Estructura de un árbol**

```
             A
          /  |  \
         B   C   D
        / \      |
       E   F     G
```

+ A es la raíz
+ B, C y D son hijos de A
+ E y F son hijos de B
+ G es hijo de D

**Conceptos fundamentales**

+ **Raíz (Root):** Es el nodo superior, Solo existe una
+ **Padre:** Nodo que tiene hijos
+ **Hijo:** Nodo descendiente de otro
+ **Hoja (Leaf):** Nodo sin hijos.
+ **Subárbol:** Cada hijo puede considerarse la raíz de un nuevo árbol.

```
        A
       / \
      B   C
     / \
    D   E
```
A -> Raíz (Root)
A -> Padre de B y C
B -> Hijo de A
C -> Hoja (Leaf)
B ->  El árbol con raíz en B es un subárbol

**Operaciones básicas**

| Operación    | Descripción                   |
| ------------ | ----------------------------- |
| Crear        | Construir un árbol vacío.     |
| Insertar     | Agregar un nodo.              |
| Eliminar     | Quitar un nodo.               |
| Buscar       | Encontrar un nodo.            |
| Recorrer     | Visitar todos los nodos.      |
| Altura       | Calcular la altura del árbol. |
| Contar nodos | Obtener el número de nodos.   |

### Recorridos

+ Los recorridos son una de las operaciones más importantes
+ Hay varias formas de recorrer un Arbol

Ejemplo:

Si tenemos el arbol:

```
        A
       / \
      B   C
     / \
    D   E
```

**Recorrido Preorden**
```
Raíz → Izquierda → Derecha
A B D E C
```

**Recorrido Inorden**
```
Izquierda → Raíz → Derecha
D B E A C
```

**Recorrido Postorden**
```
Izquierda → Derecha → Raíz
D E B C A
```

**Recorrido Por niveles**
```
A B C D E
```

## Implementación

Generalmente mediante nodos.

```
Nodo

┌──────────────┐
│ Dato         │
│ Hijos        │
└──────────────┘
```
> En un árbol general puede haber cualquier número de hijos.

## 2. Árbol Binario

Es un árbol donde cada nodo tiene como máximo dos hijos:

+ hijo izquierdo
+ hijo derecho

```
        20
       /  \
     10    35
    / \    /
   5  15  30
```
Hasta aquí no existe ninguna regla de orden.

## Árbol Binario de Búsqueda (ABB)

El ABB (Árbol Binario de Búsqueda) es una especialización del árbol binario.

Para cada nodo:

+ todos los valores del subárbol izquierdo son menores
+ todos los valores del subárbol derecho son mayores

```
           20
          /  \
        10    35
       / \    / \
      5 15  30 40
```

+ Para el nodo 20: 5, 10, 15 -> Todos menores que 20
+ Para el nodo 20: 30, 35, 40 -> Todos mayores que 20

### La propiedad se cumple recursivamente en cada subárbol.

**Búsqueda** 

Buscar 30. 

```
           20
          /  \
        10    35
             /
           30
```

Proceso:

```
30 > 20

↓

Ir a la derecha

↓

30 < 35

↓

Ir a la izquierda

↓

Encontrado
```
### No es necesario recorrer todo el árbol.

**Inserción**
```
Insertar 25.

20

↓

35

↓

30

↓

25
```
Resultado:
```
          20
         /  \
       10    35
            /
          30
         /
       25
```
**Siempre se mantiene la propiedad de orden.**

**Eliminación**

Es una de las operaciones más complejas del ABB.

Eliminar un nodo puede requerir:
* eliminar una hoja
* reemplazar un nodo con un hijo
* reemplazar un nodo con dos hijos usando su sucesor o predecesor en inorden

Si el árbol se desbalancea:

```
10
 \
 20
   \
   30
     \
     40
```
Se convierte prácticamente en una lista.

Las operaciones pasan a ser o(n)

### Por eso existen variantes balanceadas como:

+ Árbol AVL
+ Árbol Rojo-Negro
+ Árbol B
+ Árbol B+

**Comparación entre arboles**

| Característica             | Árbol                              | ABB |
| -------------------------- | ---------------------------------- | --- |
| Jerarquía                  | Sí                                 | Sí  |
| Máximo de hijos            | Ilimitado (según el tipo de árbol) | Dos |
| Regla de orden             | No                                 | Sí  |
| Optimizado para búsqueda   | No necesariamente                  | Sí  |
| Recorrido inorden ordenado | No                                 | Sí  |

## Relación entre ambos

```
Árbol
│
├── Árbol General
│
└── Árbol Binario
      │
      ├── Árbol Binario cualquiera
      │
      └── Árbol Binario de Búsqueda (ABB)
             │

```
## Variantes balanceadas ABB

Estas estructuras surgieron para resolver el principal problema del Árbol Binario de Búsqueda (ABB): puede desbalancearse.

Por ejemplo, si insertamos: 10, 20, 30, 40, 50

un ABB común puede terminar así:

```
10
  \
   20
     \
      30
        \
         40
           \
            50
```
Aunque sigue siendo un ABB, ya no es eficiente: buscar 50 requiere recorrer casi todos los nodos (O(n)).

> Las variantes balanceadas mantienen el árbol con una altura cercana a la mínima, de modo que las operaciones de búsqueda, inserción y eliminación se mantengan en O(log n).

### 1. Árbol AVL

El Árbol AVL (Adelson-Velsky y Landis) fue el primer árbol binario auto-balanceado.

Su regla es:

+ Para cada nodo, la diferencia entre la altura de sus subárboles izquierdo y derecho no puede ser mayor que 1.

**Ejemplo balanceado**
```
        30
       /  \
     20    40
    /        \
   10         50
```
> Las alturas son muy similares.

**Ejemplo desbalanceado**
```
        30
       /  
     20   
    /        
   10         
```
**El árbol debe realizar una rotación para corregirse**

Resultado:

```
      20
     /  \
   10   30
```
### Rotaciones

Existen cuatro casos clásicos:

+ Rotación simple a la derecha
+ Rotación simple a la izquierda
+ Rotación doble izquierda-derecha
+ Rotación doble derecha-izquierda

**Estas rotaciones preservan el orden del ABB mientras restauran el equilibrio.**

**Complejidad Temporal**
| Operación | Tiempo   |
| --------- | -------- |
| Buscar    | O(log n) |
| Insertar  | O(log n) |
| Eliminar  | O(log n) |

Ventajas

+ Excelente rendimiento en búsquedas
+ Árbol muy equilibrado
+ Altura mínima o cercana a la mínima


Desventajas

+ Inserciones y eliminaciones más costosas por las rotaciones frecuentes
+ Implementación relativamente compleja.

### 2. Árbol Rojo-Negro

Un Árbol Rojo-Negro también es un ABB auto-balanceado, pero utiliza un criterio de balance menos estricto.

Cada nodo tiene un color:

+ rojo
+ negro

**Ejemplo**
```
        20(B)
       /     \
   10(R)    35(B)
```
(B) = Negro

(R) = Rojo

**Reglas principales**
1. La raíz es negra
2. Todas las hojas nulas son negras
3. Un nodo rojo no puede tener un hijo rojo
4. Todo camino desde un nodo hasta una hoja contiene el mismo número de nodos negros

**¿Por qué usar colores?**
Los colores ayudan a decidir cuándo recolorear o rotar tras una inserción o eliminación, manteniendo el equilibrio sin exigir un balance perfecto como el AVL.
Estas reglas mantienen el árbol aproximadamente balanceado.

**Complejidad Temporal**
| Operación | Tiempo   |
| --------- | -------- |
| Buscar    | O(log n) |
| Insertar  | O(log n) |
| Eliminar  | O(log n) |

Ventajas
+ Menos rotaciones que un AVL
+ Inserciones y eliminaciones rápidas
+ Muy utilizado en bibliotecas estándar

Desventajas
+ El árbol puede ser un poco más alto que un AVL
+ Las búsquedas pueden ser ligeramente más lentas

**AVL vs Rojo-Negro**

**AVL**
```
Más balanceado
↓

Mejor búsqueda

Más rotaciones
```
**Rojo-Negro**
```
Menos balanceado
↓

Menos rotaciones

Inserciones más rápidas
```
### 3. Árbol B

Hasta ahora todos los árboles tenían como máximo dos hijos.

El Árbol B rompe esa limitación.

Un nodo puede contener varias claves y tener muchos hijos.

```
          [20 | 40]

        /      |      \
     <20    20-40    >40
```
**Cada nodo almacena múltiples claves ordenadas. ¿Por qué?**

En memoria secundaria (SSD, HDD), acceder a un bloque es mucho más costoso que comparar valores. Al guardar muchas claves en un mismo nodo, se reduce el número de accesos al disco.

**Ejemplo**
```

          [20 | 40 | 60]

      /      |      |      \
   ...     ...    ...     ...
```
**Ventajas**
+ Muy poca altura
+ Muy pocos accesos al disco
+ Ideal para millones de registros

**Aplicaciones**
+ Bases de datos
+ Sistemas de archivos
+ Índices en disco

### 4. Árbol B+

Es una evolución del Árbol B.

La diferencia principal es:

**Los datos completos se almacenan únicamente en las hojas.**

Los nodos internos contienen solo claves para guiar la búsqueda

**Árbol B:**

**Comparación general**
| Característica       | AVL          | Rojo-Negro | Árbol B              | Árbol B+             |
| -------------------- | ------------ | ---------- | -------------------- | -------------------- |
| Máximo de hijos      | 2            | 2          | Muchos               | Muchos               |
| Balance              | Muy estricto | Moderado   | Sí                   | Sí                   |
| Altura               | Muy baja     | Baja       | Muy baja             | Muy baja             |
| Memoria principal    | Sí           | Sí         | Principalmente disco | Principalmente disco |
| Recorrido por rangos | Bueno        | Bueno      | Bueno                | Excelente            |

**¿Cuándo usar cada uno?** 

+ AVL: cuando predominan las búsquedas y las modificaciones son menos frecuentes.
+ Rojo-Negro: cuando hay muchas inserciones y eliminaciones, manteniendo buen rendimiento general. Por eso es común en bibliotecas estándar (como TreeMap y TreeSet en Java o std::map y std::set en C++).
+ Árbol B: cuando los datos residen en disco y se quiere minimizar el número de accesos.
+ Árbol B+: cuando, además de buscar rápidamente, se realizan muchas consultas por rangos (BETWEEN, ordenaciones, recorridos secuenciales). Es la opción preferida para índices de bases de datos.

**Relación entre todos**

```
Árbol
│
└── Árbol Binario
      │
      └── ABB
           │
           ├── AVL
           └── Rojo-Negro

Árbol
│
└── Árbol Multicamino
      │
      ├── Árbol B
      └── Árbol B+
```


### Ideas clave

+ El Árbol es el TDA que modela relaciones jerárquicas entre nodos.
+ Su propósito es representar estructuras como organigramas, sistemas de archivos o árboles genealógicos.
+ El Árbol Binario de Búsqueda (ABB) es una variante especializada que añade una propiedad de orden (izquierdo < nodo < derecho).
+ Gracias a esa regla, puede localizar, insertar y eliminar elementos de forma mucho más eficiente que un árbol binario sin orden
+ La diferencia entre Arboles binarios de busqueda está en cómo mantienen el equilibrio:
    + AVL: equilibrio muy estricto mediante rotaciones
    + Rojo-Negro: equilibrio más flexible usando colores y algunas rotaciones.
    + Árbol B: equilibrio distribuyendo muchas claves e hijos en cada nodo, ideal para almacenamiento en disco.
    + Árbol B+: igual que el Árbol B, pero optimizado para índices y recorridos ordenados al almacenar los datos solo en las hojas y enlazarlas entre sí.
