# Paradigmas transversales

Existen algunos paradigmas de programación que algunos analistas consideran paradigmas por si mismos, pero aun la mayoria los consideran como un modelo de ejecución dentro de otros paradigmas.

A estos paradigmas se les suele llamar "Paradigmas transversales".

La mayoría de los lenguajes modernos son multiparadigma, permitiendo mezclar estas ideas para resolver problemas, por esto es que es importante entender un poco de estos paradigmas transversales, que estan tomando mas importancia en la industria.

A diferencia de los paradigmas principales o fundacionales (como la programación orientada a objetos o la programación funcional), estos enfoques no te obligan a cambiar la estructura base de tu código. En su lugar, se cruzan y combinan con ellos para resolver problemas específicos.

### Por qué son transversales

+ No son excluyentes: Puedes construir una aplicación utilizando el paradigma orientado a objetos (fundacional) y, al mismo tiempo, aplicar programación dirigida por eventos para la interfaz, programación paralela para acelerar los cálculos y AOP para la seguridad

+ Resuelven necesidades puntuales: No definen cómo se modela todo el software, sino cómo se maneja el tiempo, el flujo de datos, la concurrencia o la separación de funciones secundarias.
