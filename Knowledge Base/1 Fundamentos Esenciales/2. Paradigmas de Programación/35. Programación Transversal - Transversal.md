# ¿Qué es la Programación Paralela?

La programación paralela es un paradigma o modelo de programación que divide un problema en varias tareas que se ejecutan simultáneamente en diferentes unidades de procesamiento (núcleos de CPU, procesadores o incluso GPUs).

objetivos principales

+ Reducir el tiempo de ejecución aprovechando varios procesadores o núcleos al mismo tiempo.
+ Mejorar el rendimiento y reducir el tiempo de ejecución (no es la organización del código).
+ Dividir un problema grande en varios problemas pequeños para resolverlos al mismo tiempo

> Aunque suele confundirse con la programación concurrente, no son lo mismo. De hecho, la programación paralela puede considerarse una especialización de la programación concurrente
>> Es uno de los paradigmas o modelos de programación más importantes en la actualidad

**¿Cómo funciona?**

Un problema grande se divide en varias partes independientes.

*Ejemplo 1:

Calcular la suma de un arreglo con un millón de números.

habitualmente se haria asi:
```
Suma total
↓
Elemento 1
↓
Elemento 2
↓
Elemento 3...
↓
```

Con programación paralela
```
Núcleo 1
↓
Suma elementos 1–250000
----------------------

Núcleo 2
↓
Suma elementos 250001–500000
----------------------

Núcleo 3
↓
Suma elementos 500001–750000
----------------------

Núcleo 4
↓
Suma elementos 750001–1000000
```

Resultado final = Suma1 + Suma2 + Suma3 + Suma4



### Características principales

1. Ejecución simultánea

    + Varias tareas se ejecutan realmente al mismo tiempo.
    + No existe alternancia.
    + Cada núcleo ejecuta su propia tarea.

2. División del problema

    + Un problema grande se fragmenta.

3. Independencia

    + Mientras más independientes sean las tareas, mejor será el rendimiento.
    
4. Sincronización

    + Al finalizar, normalmente todas las tareas deben reunirse para producir el resultado final.

### Tipos de paralelismo

1. Paralelismo de datos (Data Parallelism)

    + Todos realizan la misma operación sobre distintos datos.
    + Todos ejecutan el mismo algoritmo.
    + Solo cambian los datos.

    > *Ejemplo 1

2. Paralelismo de tareas (Task Parallelism)

    + Cada núcleo ejecuta una tarea distinta.
    + Cada uno hace algo diferente.

    ```
    Núcleo 1
    ↓
    Procesar imágenes    
    ----------------

    Núcleo 2
    ↓
    Comprimir archivos
    ----------------

    Núcleo 3
    ↓
    Guardar información
    ```

### Arquitecturas paralelas

**Multinúcleo (Multi-Core)**

+ Una CPU con varios núcleos.
+ Es la arquitectura más común.

```
CPU

├── Core 1

├── Core 2

├── Core 3

└── Core 4
```

**Multiprocesador**

+ Varias CPUs físicas.

```
Servidor
↓
CPU 1
↓
CPU 2
↓
CPU 3
```

**GPU**

Miles de núcleos pequeños.

```
GPU

■■■■■■■■■■■■■■■■■■■■■■

Miles de núcleos
```

Ideal para:

+ Inteligencia Artificial.
+ Renderizado.
+ Simulaciones.
+ Criptografía.

### Ventajas

**Mayor velocidad**
+ Es la ventaja principal: Un trabajo puede tardar segundos en lugar de minutos.

**Mejor aprovechamiento del hardware**
+ Los núcleos trabajan simultáneamente.

**Escalabilidad**

Puede aprovechar:
+ 4 núcleos.
+ 8 núcleos.
+ 32 núcleos.
+ Cientos de procesadores.

**Grandes volúmenes de datos**

Ideal para:

+ Machine Learning.
+ Big Data.
+ Simulaciones.
+ Videojuegos.
+ Modelado científico.

### Desventajas

**Programación más compleja**
+ Dividir correctamente un problema no siempre es sencillo.

**Sincronización**
+ Las tareas deben coordinarse.

**Comunicación**
+ A veces los núcleos necesitan compartir información, Eso genera sobrecarga.

**No todos los problemas son paralelizables**

Ejemplo:
```
Paso 1
↓
Paso 2
↓
Paso 3
↓
Paso 4
```
**Si el paso 2 depende del 1, no pueden ejecutarse al mismo tiempo.**

### Concurrencia vs Paralelismo

Esta comparación es una de las más importantes.

| Concurrencia                             | Paralelismo                                            |
| ---------------------------------------- | ------------------------------------------------------ |
| Varias tareas progresan al mismo tiempo. | Varias tareas se ejecutan físicamente al mismo tiempo. |
| Puede existir un solo núcleo.            | Requiere varios núcleos o procesadores.                |
| Se centra en organizar múltiples tareas. | Se centra en acelerar la ejecución.                    |
| No siempre mejora el rendimiento.        | Busca mejorar el rendimiento.                          |
| Puede alternar entre tareas.             | Las tareas ocurren simultáneamente.                    |

**Ejemplo práctico**

Supongamos que debes procesar 1 millón de fotografías.

**Concurrencia**

```
CPU
↓
Foto 1
↓
Foto 2
↓
Foto 3
↓
Foto 4
```
La CPU cambia rápidamente entre tareas.

**Paralelismo**

```
CPU 1 → Foto 1

CPU 2 → Foto 2

CPU 3 → Foto 3

CPU 4 → Foto 4
```

Todas avanzan simultáneamente.

**¿Por qué surgió?**

1. Durante muchos años, los procesadores aumentaban su velocidad (frecuencia) para ejecutar programas más rápido.
2. Sin embargo, llegó un punto en el que aumentar la frecuencia generaba demasiado calor y consumo de energía.
3. La solución fue fabricar procesadores con varios núcleos.
4. Ahora el reto ya no era construir CPUs más rápidas, sino escribir programas capaces de aprovechar varios núcleos.
5. Ahí nace la programación paralela.

### Lenguajes con soporte para programación paralela

| Lenguaje | Herramientas de paralelismo                                         |
| -------- | ------------------------------------------------------------------- |
| C/C++    | OpenMP, MPI, CUDA                                                   |
| Java     | Fork/Join Framework, Parallel Streams                               |
| C#       | Task Parallel Library (TPL), Parallel LINQ (PLINQ)                  |
| Python   | multiprocessing, concurrent.futures                                 |
| Go       | Goroutines (pueden ejecutarse en paralelo si hay varios núcleos)    |
| Rust     | Rayon, Tokio (para tareas concurrentes y, según el caso, paralelas) |

> Muchos lenguajes modernos ofrecen herramientas específicas para aprovechar el hardware paralelo

### Casos de uso

La programación paralela es especialmente útil cuando el problema puede dividirse en partes independientes:

+ **Inteligencia Artificial:** entrenamiento de redes neuronales.
+ **Big Data:** procesamiento distribuido de grandes volúmenes de información.
+ **Renderizado 3D:** cálculo simultáneo de píxeles o regiones de una imagen.
+ **Videojuegos:** física, inteligencia artificial y renderizado ejecutándose en paralelo.
+ **Simulaciones científicas:** clima, dinámica molecular o ingeniería.
+ **Procesamiento de imágenes y video:** aplicar filtros o transformaciones a miles de elementos al mismo tiempo

### Relación con otros paradigmas

La programación paralela no sustituye a paradigmas como la programación orientada a objetos o la funcional; los complementa. Por ejemplo:

+ **POO + Paralelismo:** una aplicación Java que utiliza Parallel Streams.
+ **Funcional + Paralelismo:** operaciones sobre colecciones que se ejecutan en varios núcleos.
+ **Imperativa + Paralelismo:** un programa en C con OpenMP.
+ **Concurrente + Paralelismo:** un servidor que gestiona múltiples clientes y además distribuye cálculos entre varios núcleos.

### Conclusión

La programación paralela representa un paso más allá de la programación concurrente. Mientras la concurrencia se enfoca en gestionar múltiples tareas que avanzan de forma coordinada, la programación paralela busca que esas tareas se ejecuten realmente al mismo tiempo para aprovechar el hardware moderno y reducir el tiempo de procesamiento.

En la práctica, ambos conceptos suelen trabajar juntos: un sistema concurrente puede distribuir trabajo entre varios núcleos y, cuando el hardware lo permite, ese trabajo se ejecuta en paralelo. Por eso, comprender primero la concurrencia y después la programación paralela proporciona una base sólida para desarrollar aplicaciones modernas de alto rendimiento.
