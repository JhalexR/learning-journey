# Paradigmas de Programación

Los paradigmas de programación son diferentes formas de pensar y estructurar la solución de un problema mediante código

Cada paradigma propone una manera distinta de organizar los programas, los datos y la lógica.

### 1. Programación Imperativa

+ Describe cómo hacer algo paso a paso.
+ es una secuencia de instrucciones que modifican el estado
+ una secuencia de instrucciones que la computadora ejecutará en el orden indicado.

El programador controla explícitamente:

+ Variables
+ Asignaciones
+ Condicionales
+ Ciclos
+ Cambios de estado

ejemplo: 

Si queremos sumar los números del 1 al 5:

```
suma = 0

para i desde 1 hasta 5
    suma = suma + i

mostrar suma
```

Aquí le estamos diciendo a la computadora exactamente cómo hacerlo.

> Es el paradigma más intuitivo y la base sobre la que se construyen los demás.

**Características**

+ Paso a paso.
+ Uso intensivo de variables.
+ El estado cambia constantemente.
+ Muy eficiente para muchos problemas.

### 2. Programación Orientada a Objetos (POO)

Organiza el código alrededor de *objetos* que combinan datos

Un *objeto* representa una entidad que posee:

+ `Datos` (atributos)
+ `Comportamientos` (métodos)

Por ejemplo:

```
Objeto: Automóvil

Atributos:
- color
- marca
- velocidad

Métodos:
- acelerar()
- frenar()
- encender()
```
**En lugar de pensar en pasos, pensamos en entidades que interactúan.**

**Principios fundamentales**

+ Clases e instancias: plantillas y objetos concretos.
+ Encapsulamiento: ocultar detalles, exponer interfaz pública.
+ Herencia: crear clases basadas en otras (usar con moderación).
+ Polimorfismo: mismo mensaje, diferentes comportamientos.
+ Abstracción: modelar solo lo relevante del dominio.

**Ventajas**

+ Código organizado.
+ Reutilización.
+ Fácil mantenimiento.
+ Escalable para proyectos grandes.

### 3. Programación Funcional

Trata la computación como evaluación de funciones matemáticas.

Una función recibe datos y devuelve un resultado sin modificar nada fuera de ella.

> Produce código más predecible y testeable

En este paradigma se intenta evitar:

+ Variables mutables
+ Estados compartidos
+ Efectos secundarios

Ejemplo
```
doble(x)

Entrada: 5

Salida: 10
```

La función siempre produce el mismo resultado para la misma entrada.

**Características**

+ Funciones puras: misma entrada = misma salida, sin efectos secundarios.
+ Inmutabilidad: los datos no cambian, se crean nuevas versiones.
+ Funciones de orden superior: funciones como argumentos o retornos.
+ Map, Filter, Reduce: transformaciones declarativas de colecciones.
+ Composición: combinar funciones simples para crear complejas.
+ Recursión en lugar de bucles (en muchos casos).

**Ventajas**

+ Menos errores.
+ Código más predecible.
+ Facilita el paralelismo.

### 4. Programación Declarativa

En este paradigma el programador describe qué desea obtener, pero no especifica el algoritmo detallado para conseguirlo.

El sistema se encarga de encontrar la solución.

Ejemplo

SQL:
```SQL
SELECT *
FROM Estudiantes
WHERE Edad > 18;
```
> No indicamos cómo buscar los registros.

**Características**

+ Se centra en el resultado.
+ Menor cantidad de código.
+ Alto nivel de abstracción.

### Resumen

| Paradigma                                  | Idea principal                                                                  | Pregunta que responde             |
| ------------------------------------------ | ------------------------------------------------------------------------------- | --------------------------------- |
| **Programación Imperativa**                | Indica paso a paso cómo resolver un problema.                                   | **¿Cómo hacerlo?**                |
| **Programación Orientada a Objetos (POO)** | Organiza el programa mediante objetos que representan entidades del mundo real. | **¿Quién realiza la acción?**     |
| **Programación Funcional**                 | Construye programas mediante funciones puras y evita modificar el estado.       | **¿Qué transformación realizar?** |
| **Programación Declarativa**               | Describe el resultado esperado sin especificar el procedimiento.                | **¿Qué quiero obtener?**          |

**Una analogía para entenderlos**

Imagina que quieres preparar una taza de café:

+ **Programación Imperativa:** Escribes la receta paso a paso: "calienta agua, agrega café, vierte el agua, mezcla y sirve".
+ **Programación Orientada a Objetos:** Modelas los elementos como objetos: `una Cafetera`, una `Taza` y un `Usuario`, donde cada uno tiene *atributos y métodos* (`preparar()`, `servir()`, etc.).
+ **Programación Funcional:** Piensas en una serie de transformaciones: agua → agua caliente → café preparado → café servido, evitando modificar los datos originales.
+ **Programación Declarativa:** Solo expresas el objetivo: *"quiero una taza de café".* El sistema decide los pasos necesarios para prepararla.

### Conclusión

**Estos paradigmas no son excluyentes. De hecho, la mayoría de los lenguajes modernos son multiparadigma**
